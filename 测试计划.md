# P0 测试计划

## 1. 文档目标

本文定义 Little Bear RAG 后端第一阶段 MVP 的 P0 测试计划，作为正式编码、测试实现、CI 门禁、缺陷回归和验收评审的共同输入。

测试目标不是追求形式覆盖率，而是验证以下 P0 闭环在工程上可靠成立：

```text
初始化系统 -> 登录 -> 组织/用户/权限 -> 知识库 -> 文档导入 -> 索引发布 -> 权限过滤查询 -> 带引用答案 -> 审计可追踪
```

## 3. 测试原则

- P0 自动化测试必须能使用模型 mock 或单一模型 adapter 跑通，不依赖真实大模型。
- OpenAPI 是 API contract 测试、后端实现和联调验收的唯一接口契约。
- PostgreSQL 是状态事实源；Redis、Qdrant、关键词索引、对象存储和模型网关均按派生状态或外部 adapter 测试。
- 高风险权限、删除阻断、权限收紧、token 越权、缓存串权和 citation 校验必须自动化。
- 查询链路按 fail closed 验证：不确定权限、索引版本、文档状态、chunk 状态或 access block 时必须拒绝或返回无结果。
- 测试数据必须可重复构造和清理；外部依赖中的测试 collection、对象 key、缓存 key 必须带 `test_run_id`。
- 测试日志、断言输出、审计摘要和模型调用日志不得包含 password、token、secret value、完整 prompt 或未脱敏敏感原文。
- 不为 P1/P2 能力编写 P0 门禁测试，例如外部组织同步、完整审批流、复杂多 Agent 编排、完整离线评测平台或真实大模型高可用调度。

## 4. 测试分层

| 层级 | 目标 | P0 覆盖范围 | 自动化要求 |
| --- | --- | --- | --- |
| policy 单元测试 | 验证纯规则、纯函数和安全边界 | 权限策略、资源可见性、缓存 key、citation 准入、脱敏规则、配置安全规则、状态迁移判断 | 必须自动化，禁止访问数据库和外部服务 |
| service 用例测试 | 验证业务用例、事务边界和异常分支 | setup、auth、org、permission、config、import、index、query、audit | 必须自动化，可使用 repository fake 或测试数据库 |
| repository 数据库测试 | 验证 schema、约束、索引、CAS 状态迁移和查询过滤 | migration、唯一约束、外键、软删除、access block、权限快照、跨租户过滤 | 必须自动化，使用测试 PostgreSQL |
| adapter 集成测试 | 验证外部依赖封装和超时清理 | Redis、MinIO、Qdrant、PostgreSQL FTS、Secret Store、Model Gateway mock | 必须自动化，可在本地 docker compose 或 CI service 中运行 |
| API contract 测试 | 验证 OpenAPI 契约和网关行为 | request/response schema、错误结构、HTTP status、鉴权声明、分页、SSE、multipart 上传 | 必须自动化，以 `openapi.yaml` 为准 |
| 端到端测试 | 验证完整用户路径 | 初始化、管理员配置、文档导入、权限查询、越权阻断、删除阻断、模型 mock 降级 | P0 主路径必须自动化 |
| 权限绕过测试 | 验证攻击和误用路径 | token 类型错用、scope 缺失、跨部门、跨租户、缓存复用、旧权限版本、setup JWT 越权 | 必须自动化，作为 PR 阻断门禁 |
| 超时、重试、熔断和降级测试 | 验证不稳定依赖下的可控行为 | embedding、rerank、LLM mock、Qdrant、关键词索引、对象存储、Worker lock | 必须自动化，至少覆盖 P0 降级策略 |

## 5. 测试环境

P0 最小测试环境：

- PostgreSQL
- Redis
- MinIO 或对象存储 mock adapter
- Qdrant
- PostgreSQL Full Text / 关键词检索能力
- Secret Store mock
- Model Gateway mock
- Embedding mock
- Rerank mock
- LLM mock
- API Service
- Worker

环境约束：

- 进程启动时只允许读取数据库连接配置和进程参数。
- 业务配置必须通过初始化接口写入 `active_config v1`。
- 未初始化时只允许 setup 只读/写接口和 healthcheck 中符合生命周期门禁的接口。
- 每个测试进程必须生成独立 `test_run_id`，用于数据库命名空间、对象 key 前缀、Qdrant collection 后缀、Redis key 前缀和日志筛选。
- 测试结束必须清理数据库记录、对象存储测试对象、Qdrant 测试 collection、Redis key 和 mock 调用记录。

## 6. 基础测试数据

P0 端到端和权限测试统一使用以下最小数据集：

| 数据 | 说明 |
| --- | --- |
| 默认企业 | 初始化创建，P0 不测试多 active enterprise |
| 默认部门 | 初始化创建 |
| 销售部门 | 管理员配置验收创建 |
| 管理员 | 初始化创建首个系统管理员 |
| Alice | 销售部门普通用户，拥有查询销售知识库权限 |
| Bob | 非销售部门普通用户，用于越权阻断测试 |
| 销售知识库 | 默认 `department` 可见 |
| 销售政策文档 | 含固定文本、页码和 chunk，可生成确定性 citation |
| 模型 mock 响应 | embedding、rerank、LLM 输出固定，支持按测试注入 timeout、error 和 degraded |

固定文档内容必须满足：

- 能被关键词检索命中。
- 能被 embedding mock 生成确定性向量。
- 切块结果数量可预测。
- citation 能回溯到 `document_version_id`、`chunk_id`、页码和对象存储 key。

## 7. P0 必测场景矩阵

`automation_required=yes` 的用例必须进入 CI 门禁；`manual_allowed=yes` 只表示允许保留人工复核步骤，不表示可以跳过自动化主断言。

| ID | 场景 | 测试层级 | gate | automation_required | manual_allowed | 关键断言 |
| --- | --- | --- | --- | --- | --- | --- |
| P0-SETUP-001 | 首次初始化成功 | API contract、service、repository、E2E | e2e | yes | no | 空库返回未初始化；配置 schema 校验通过；创建管理员、默认企业、默认部门、默认角色和 `active_config v1`；`GET /health/ready` ready；写入 `setup.initialized` 审计 |
| P0-SETUP-002 | 初始化生命周期门禁 | API contract、权限绕过、E2E | security | yes | no | 未初始化时普通业务、管理后台和服务间 API 返回 `SETUP_REQUIRED`；初始化完成后 setup 写接口返回 `SETUP_CLOSED`；不得再次写入 active config |
| P0-SETUP-003 | setup JWT CLI 签发和轮换 | CLI contract、service、repository、权限绕过 | security | yes | yes | `rag-admin setup-token issue/rotate/revoke` 每次签发新 token 都使旧 active token 失效；系统内最多一个 active setup token；JWT 明文只展示一次；写入 setup token 审计 |
| P0-SETUP-004 | setup JWT 越权阻断 | API contract、权限绕过 | security | yes | no | setup JWT 不能访问普通业务 API、管理员 API 或服务间 API；写入 `auth.token_scope_denied` 审计 |
| P0-CONFIG-001 | active config 读取和版本查询 | API contract、service、repository | contract | yes | no | 管理员可读取 active config 摘要和 config version；响应只返回 Secret 引用和脱敏摘要；无 `config:read` 被拒绝 |
| P0-CONFIG-002 | 配置校验和发布 | API contract、service、repository、adapter | e2e | yes | no | 配置保存 draft、校验 schema 和依赖、发布 active version；发布失败时 active 指针不变；critical 审计写入 |
| P0-AUTH-001 | 本地账号登录成功和失败锁定 | API contract、service、repository | e2e | yes | no | 正确密码签发 access/refresh JWT；错误密码增加失败计数；达到阈值锁定；失败审计不泄露密码 |
| P0-AUTH-002 | refresh token rotation | API contract、service、repository、权限绕过 | security | yes | no | refresh 成功后旧 refresh token 变为 `used`；同一 refresh token 二次使用被拒绝并审计；access token 不可调用 refresh endpoint |
| P0-AUTH-003 | 当前用户、改密和登出 | API contract、service、repository | e2e | yes | no | 当前用户信息不泄露敏感字段；当前用户改密需要二次确认并撤销旧会话；登出后旧 access token 不可继续使用 |
| P0-AUTH-004 | 管理员禁用、软删除、重置密码和解锁 | service、repository、E2E | security | yes | no | 禁用或软删除用户后旧 access/refresh token 不可继续使用；重置密码和解锁写审计；权限缓存失效 |
| P0-ORG-001 | 管理员创建用户、部门和角色绑定 | API contract、service、repository、E2E | e2e | yes | no | Alice 加入销售部门并绑定普通员工角色；Permission Service 可构建权限上下文；角色绑定写审计 |
| P0-PERM-001 | 越界权限策略拒绝 | policy、API contract、权限绕过 | security | yes | no | P0 只接受 `department` 和 `enterprise` 可见性；请求中出现用户级 ACL、项目组或自定义组织 ACL 返回 `UNSUPPORTED_PERMISSION_POLICY` |
| P0-KB-001 | 知识库、文件夹和文档元数据 | API contract、service、repository | e2e | yes | no | 创建销售知识库、文件夹、文档和版本；disabled/archived/deleted 资源不能被普通查询命中 |
| P0-IMPORT-001 | 文件上传创建导入任务 | API contract、service、repository、E2E | e2e | yes | no | 上传后创建 document、document_version、import_job；任务初始状态为 `queued`；对象存储写入成功；审计记录导入开始 |
| P0-IMPORT-002 | URL 或批量元数据创建导入任务 | API contract、service、repository | integration | yes | no | 创建 import_job；复用同一权限、状态机和审计规则；批量部分失败可进入 `partial_success` |

| P0-INDEX-001 | draft index 查询不可见 | repository、adapter、E2E、权限绕过 | security | yes | no | draft chunk、draft index、blocked index ref 不被查询命中；发布前校验 draft 不可见 |
| P0-INDEX-002 | active index 查询可见 | repository、adapter、E2E | e2e | yes | no | active index 发布成功后 Alice 可召回销售政策 chunk；旧索引进入 archived 或 pending_delete |
| P0-INDEX-003 | 发布前一致性校验 | service、repository、adapter | integration | yes | no | chunk 数量、向量维度、payload hash、权限快照和 index version 不一致时拒绝发布；旧 active index 继续服务 |
| P0-QUERY-001 | 权限过滤和候选准入 | policy、adapter、E2E、权限绕过 | security | yes | no | 向量召回和关键词召回均下推 enterprise、department、document status、chunk status、active index 等可索引过滤；候选进入上下文前必须校验 active access block 和 citation 来源 |
| P0-QUERY-002 | 查询返回 citation | API contract、service、E2E | e2e | yes | no | 返回答案和 citation；citation 可回溯到 active chunk、document version、页码和对象存储原文 |
| P0-QUERY-003 | strict citation 流式响应 | API contract、service、E2E | e2e | yes | no | 严格引用模式下不得输出未经引用校验的最终 token；citation 校验失败时抑制最终答案并审计 |
| P0-PERM-002 | Bob 查询销售部门文档被阻断 | policy、adapter、E2E、权限绕过 | security | yes | no | Bob 不召回销售部门可见文档；无结果或权限拒绝响应不泄露文档存在性 |
| P0-PERM-003 | 文档可见性扩大 | service、repository、E2E | e2e | yes | no | `department -> enterprise` 后 Bob 可查询该文档；权限扩大写审计；查询缓存失效 |
| P0-PERM-004 | 权限收紧后立即阻断 | service、repository、adapter、E2E、权限绕过 | security | yes | no | `enterprise -> department` 时先写 access block；Bob 不能通过旧缓存、旧索引或旧 permission_version 访问；新受限 refs 发布后保持正确可见性 |
| P0-DELETE-001 | 文档删除阻断 | service、repository、adapter、E2E、权限绕过 | security | yes | no | 删除先写 access block；即使 Qdrant 或关键词索引物理删除失败，Alice 也不能查询到该文档；清理失败进入重试并审计 |
| P0-CACHE-001 | 查询缓存隔离 | policy、service、E2E、权限绕过 | security | yes | no | 缓存 key 包含用户、企业、权限过滤、权限版本、索引版本、请求过滤条件和配置版本；不得跨用户、跨租户、跨权限复用 |
| P0-DEGRADE-001 | Embedding mock 不可用 | adapter、service、E2E | e2e | yes | no | 查询降级为关键词检索；仍携带权限过滤；响应、查询日志、模型调用日志和审计记录降级原因 |
| P0-DEGRADE-002 | Rerank mock 不可用 | adapter、service、E2E | e2e | yes | no | 查询使用融合分数排序；不丢失权限过滤和 citation 校验 |
| P0-DEGRADE-003 | LLM mock 不可用 | adapter、service、E2E | e2e | yes | no | 返回检索结果和 citation，不输出无引用答案；写 `model_call.failed` 和 `query.degraded` |
| P0-AUDIT-001 | 审计和脱敏 | service、repository、E2E | security | yes | no | 初始化、登录、用户管理、角色绑定、权限变更、配置发布、导入、删除、查询降级和模型调用写入正确事件；摘要不含禁止字段 |
| P0-AUDIT-002 | 审计、查询日志和模型调用日志查询 | API contract、service、repository | contract | yes | yes | `audit:read` 用户可按用户、知识库、状态、是否降级、配置版本和权限版本筛选日志；普通用户不可访问；响应脱敏 |
| P0-CONTRACT-001 | OpenAPI contract 回归 | API contract | contract | yes | no | 所有 P0 endpoint 的 request、response、error schema、status code、鉴权声明和分页/SSE/multipart 契约与 `openapi.yaml` 一致 |

## 8. 状态机测试要求

状态机测试必须覆盖合法迁移、非法迁移、幂等、失败重试、取消和补偿阻断。

P0 必测状态机：

- setup 初始化状态机。
- setup JWT 状态机。
- access / refresh token 状态机。
- enterprise、department、user、membership、role binding 状态机。
- import_job 状态机。
- knowledge_base 与 folder 状态机。
- document lifecycle、document index 和 visibility 状态机。
- document_version 状态机。
- chunk 生命周期状态机。
- index_version 状态机。
- chunk_index_ref 与 keyword_index_entry 可见性状态机。
- config_version 状态机。
- access_block 生命周期。

状态机测试必须断言：

- 所有状态变更通过显式 transition 方法执行。
- compare-and-set 条件不满足时不会静默成功。
- 高风险状态变更和审计记录在同一事务提交。
- 终态默认不可逆；重新处理优先创建新版本或新任务。
- 删除、权限收紧、权限版本落后和索引不一致场景 fail closed。

## 9. API Contract 测试要求

API contract 测试以 `openapi.yaml` 为准，至少覆盖：

- 所有 P0 endpoint 的 HTTP method、path、query、path params、request body 和 response body。
- 标准错误结构：`request_id`、`error_code`、`message`、`retryable`。
- 鉴权类型：none、setup token、access token、refresh token、service token。
- `x-required-scope` 与 `权限矩阵.md` 一致。
- setup 生命周期门禁。
- multipart 文件上传。
- SSE 查询流事件 schema。
- 分页字段和过滤字段。
- 高风险写接口的 `x-confirmation-required`、`x-confirmation-condition` 和审计声明。

任何 OpenAPI 非兼容变更都必须同时更新：

- API 实现。
- 权限矩阵。
- 审计事件字典。
- P0 测试用例。
- 前端或 SDK 契约。

## 10. 权限绕过测试要求

以下场景必须自动化，且失败时阻断合并：

- 未初始化时访问普通业务 API。
- setup JWT 访问普通业务 API、管理员 API 或服务间 API。
- refresh token 访问普通业务 API。
- access token 调用 refresh endpoint。
- 无 scope、错 scope 或过期 token 访问受保护接口。
- 普通用户调用管理员接口。
- Bob 访问销售部门可见知识库、文档、chunk、导入任务或 citation。
- 文档从 `enterprise` 收紧为 `department` 后，Bob 使用旧缓存、旧索引或旧 permission_version 查询。
- draft index、deleted document、blocked document、archived index 被查询命中。
- 查询缓存跨用户、跨企业、跨权限过滤条件、跨权限版本或跨索引版本复用。
- API、日志、审计摘要、模型调用日志泄露 password、token、secret value、完整 prompt 或未脱敏敏感原文。

## 11. 降级和故障注入测试要求

P0 故障注入使用 mock adapter 或可控测试开关实现，禁止依赖随机网络故障。

| 依赖 | 故障 | 期望行为 |
| --- | --- | --- |
| Embedding mock | timeout、5xx、维度不匹配 | 查询降级为关键词检索或返回结构化失败；导入索引失败进入可重试状态 |
| Rerank mock | timeout、5xx、空结果 | 使用融合分数或原始召回排序；保留 citation 校验 |
| LLM mock | timeout、5xx、返回无 citation 内容 | 返回检索结果和 citation 或抑制最终答案；不得输出无引用答案 |
| Qdrant | 写入失败、删除失败、查询超时 | 写入失败不发布 active index；删除失败时 access block 保持 active；查询可按策略降级或失败 |
| 关键词索引 | 写入失败、删除失败、查询超时 | 写入失败不发布 active index；删除失败时 access block 保持 active；查询可按策略降级或失败 |
| MinIO | 上传失败、读取失败、删除失败 | import_job 失败或重试；预览失败返回结构化错误；删除失败不释放 access block |
| Redis | 缓存不可用、锁不可用 | 查询可绕过缓存；分布式锁失败时拒绝高风险并发操作或进入重试 |
| Audit Repository | 写入失败 | 高风险写接口返回 `AUDIT_WRITE_FAILED` 或触发告警，不得静默成功 |

## 12. 自动化门禁

建议将 P0 测试拆成以下执行集合：

| 门禁 | 触发时机 | 范围 |
| --- | --- | --- |
| fast | 每次提交或 PR | policy 单元测试、service 用例测试、OpenAPI lint、脱敏规则测试 |
| db | PR | migration、repository、状态机 CAS、约束和索引测试 |
| contract | PR | API contract、鉴权声明、错误结构、分页、SSE、multipart schema |
| integration | PR 或合并前 | Redis、MinIO、Qdrant、关键词索引、Model Gateway mock adapter |
| e2e | 合并前和 nightly | 初始化到权限过滤查询完整闭环、删除阻断、模型 mock 降级 |
| security | PR 阻断 | 权限绕过、token 越权、缓存串权、敏感信息泄露 |

合并进入主干前必须满足：

- P0 必测场景矩阵中 `automation_required=yes` 的用例全部通过，且对应 gate 已纳入 CI。
- 高风险权限场景全部自动化且通过。
- OpenAPI contract 测试通过。
- 数据库 migration 可以从空库执行成功。
- 测试环境不依赖真实大模型。
- 测试运行后外部依赖测试数据可清理。

## 13. 手工验收边界

P0 手工验收只用于补充以下内容，不得替代高风险自动化测试：

- 本地开发环境首次启动文档是否可执行。
- healthcheck、ready、日志、Trace 和审计查询的人工检查。
- 管理后台或临时 API 客户端的操作体验确认。
- Worker 和 API 多进程启动顺序验证。

以下内容不得只靠手工验收：

- 权限绕过。
- token 越权。
- 权限收紧。
- 删除阻断。
- 查询缓存隔离。
- strict citation 输出控制。
- 脱敏和禁止字段检查。

## 14. 缺陷回归要求

每个 P0 缺陷修复必须新增或更新回归测试。回归测试至少说明：

- 触发条件。
- 复现步骤。
- 涉及 API、service、repository 或 adapter。
- 修复后的关键断言。
- 是否涉及权限、安全、审计、状态机或数据一致性。

权限、安全、删除阻断、审计和数据泄露类缺陷必须进入 `security` 或 `e2e` 门禁集合。

## 15. 失败定位要求

P0 自动化测试失败时，测试报告至少应能定位：

- `request_id` 或 `trace_id`。
- API path、method、status code 和 `error_code`。
- actor、enterprise、department、role、scope 和 token type 摘要。
- 关键资源 ID：knowledge_base、document、document_version、chunk、index_version、import_job。
- 状态机 from/to 和失败原因。
- adapter 调用摘要、耗时、超时或降级原因。
- 审计事件名、result、risk_level 和脱敏 summary。

失败排查顺序：

1. 确认测试夹具和初始化状态。
2. 查看 API 错误结构和 `request_id`。
3. 查看状态事实源表。
4. 查看权限上下文和过滤条件。
5. 查看 adapter mock 调用记录。
6. 查看审计日志、查询日志和模型调用日志。
7. 确认外部依赖测试数据是否清理。

## 16. 完成标准

- 每条 P0 链路都有自动化测试或明确手工验收步骤。
- 高风险权限、删除阻断、token 越权、缓存串权和敏感信息泄露场景必须自动化。
- 测试数据可重复构造和清理。
- 本地和 CI 均可使用模型 mock 跑通初始化、认证、权限、导入、索引、查询、删除阻断和模型 mock 降级。
- 测试计划能映射到 `openapi.yaml`、`database-schema.md`、`权限矩阵.md`、`状态机设计.md` 和 `审计事件字典.md`。

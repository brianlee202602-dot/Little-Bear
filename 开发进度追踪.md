# Little Bear 开发进度追踪

更新时间：2026-05-16 15:27:22 CST

本文用于记录当前项目实现进度，便于后续按 MVP 闭环持续追踪开发状态。详细设计以 `docs/` 下架构文档、OpenAPI、数据库 Schema、权限矩阵、状态机和审计事件字典为准。

## 1. 当前阶段判断

当前项目已经从“工程骨架”推进到“控制面功能初步可用，并具备 RAG 数据面最小链路”的阶段。

已具备：

- 初始化、认证、配置、管理后台用户/部门/角色绑定、审计查询等控制面能力。
- Permission Service 核心能力已落地，包括权限上下文、统一过滤条件、候选准入 gate 和可见性策略校验。
- 数据库迁移已覆盖 P0 大部分核心表。
- 管理后台前端已接入 setup、登录、配置、用户、部门、角色、审计等接口。
- 已新增 OpenAPI 与实际 FastAPI routes 的契约差异测试。
- Import Service、Worker 和 Indexing Service 最小链路已落地，支持上传 / URL / metadata_batch 导入任务创建、任务查询、取消、重试、Worker claim、阶段推进、draft chunk 写入、PostgreSQL keyword index、Qdrant draft vector point 写入和 active 发布。
- Query Service 非流式最小链路已落地，支持 `POST /internal/v1/queries`、关键词召回、query embedding client、Qdrant VectorRetriever adapter、RRF 融合排序、rerank provider、Permission Service filter、候选 gate、Context Builder、LLM provider、citation 校验、query_logs、model_call_logs 和高风险 query audit 写入；rerank、LLM 不可用或 citation 校验失败时结构化降级为 retrieval-only 检索来源。
- Query Stream 第一版已落地，支持 `POST /internal/v1/query-streams` SSE 输出；当前采用服务端先完成合规查询与 citation 校验，再输出 metadata/token/citation/done 事件。
- 普通用户 Web 查询工作区第一版已落地，支持登录、token refresh、退出登录、知识库浏览、文档浏览、citation 来源跳转、查询模式、流式/非流式调用、降级状态、request_id 和 trace_id 展示。
- 已新增 P0 主链路 smoke 脚本和 `make smoke-p0` 入口，覆盖登录、当前用户、知识库浏览、文档/来源浏览、非流式查询和 SSE 查询。
- `employee` 内置角色已补齐 `knowledge_base:read`，并新增存量角色迁移，避免普通员工登录后无法进入知识库浏览。
- 导入链路已接入 MinIO/S3 兼容对象存储 runtime，API 上传与 Worker 可通过 `active_config.storage` 共享原始对象；parse / clean / chunk 第一版执行器已落地，支持 PDF、DOCX、UTF-8 文本和 Markdown。
- 管理后台已将“文档导入”调整为“知识库管理”，支持知识库列表、新增、编辑、删除，并可从指定知识库弹窗添加文本 / Markdown 文件、查看导入任务状态。
- 已完成一次真实 Markdown 上传端到端联调：管理后台上传 -> Import Job -> Worker -> parse / clean / chunk -> embedding -> Qdrant 写入 -> active index 发布；已确认文档、版本、chunks、index refs 和 Qdrant 文档 points 均为 active。
- 后端单元测试、普通 Web 与管理后台 typecheck/build 当前通过。

尚未具备：

- 复杂版式 / OCR 解析、查询改写和真实模型 token 级流式输出等完整 RAG 数据面闭环。

MVP 目标闭环仍是：

```text
初始化系统 -> 登录 -> 组织/用户/权限 -> 知识库 -> 文档导入 -> 索引发布 -> 权限过滤查询 -> 带引用答案 -> 审计可追踪
```

## 3. 剩余未完成与待加固功能

### 3.1 Permission Service

Permission Service 核心已实现，并已接入知识库浏览、导入索引和查询主链路：

- 已实现：
  - 构建 `PermissionContext`。
  - 生成统一 `PermissionFilter`。
  - 为 Qdrant payload、关键词检索 SQL、元数据查询 SQL 输出同一套权限语义。
  - 查询后候选准入 gate，覆盖企业、知识库、文档状态、chunk 状态、active index、access block、权限版本和可见性。
  - P0 可见性策略校验，拒绝指定用户、项目组、自定义 ACL、递归部门等非 P0 能力。
  - 权限快照 payload hash 生成。
  - 知识库、文件夹、文档元数据、导入和查询 API 使用 Permission Service 做范围过滤或候选准入。
  - 管理端文档元数据权限字段变更会写 resource policy / permission snapshot。
  - 文档删除会先写 access block，避免旧索引继续被访问。
  - 查询链路会记录权限版本、候选拒绝和高风险权限审计事件。
- 仍缺少：
  - 独立文档权限变更 API 的完整产品面。
  - 权限收紧后的索引 payload 刷新 / 重建任务闭环。
  - 权限缓存失效、旧索引清理和权限收紧 fail closed 的系统级回归测试。

### 3.3 导入与 Worker

Import Service 与 Worker 已完成第一版最小链路：

- `POST /internal/v1/knowledge-bases/{kb_id}/document-imports`
- `GET /internal/v1/import-jobs/{job_id}`
- `PATCH /internal/v1/import-jobs/{job_id}`
- `POST /internal/v1/import-jobs/{job_id}/retries`
- `GET /internal/v1/admin/import-jobs`
- `GET /internal/v1/admin/import-jobs/{job_id}`
- 上传 / URL / metadata_batch 导入时创建 `documents`、`document_versions`、`permission_snapshots` 和 `import_jobs`。
- 导入任务支持幂等键、查询、取消和手动重试。
- Worker 基于 `import_jobs`、`locked_by`、`locked_until` 领取 queued / retrying 任务。
- Worker 持有锁后会连续推进同一任务，避免任务停在 running 中间阶段。
- Worker 支持阶段推进：
  - validate
  - parse
  - clean
  - chunk
  - embed
  - index
  - publish
  - cleanup
  - finished
- Worker 支持 retrying / failed / cancelled / success 状态落库。
- 初期 parse / clean / chunk 支持纯文本或 Markdown 内容，并写入 draft `chunks`。

仍缺少：

- 锁续约的独立定时 heartbeat 与崩溃接管集成测试。
- 扫描件 OCR、复杂版式解析、表格结构保真和解析质量评估。
- 真实文件解析后的 chunk 质量控制和失败补偿。
- dead-letter 查询与运维入口。

### 3.4 索引服务

已完成第一版最小链路：

- 创建 `index_versions(status=draft)`。
- PostgreSQL Full Text draft 关键词索引写入。
- `chunk_index_refs` 账本写入，使用 Qdrant 兼容的确定性 UUID `vector_id`。
- ModelGateway embedding client 支持 batch embedding。
- Qdrant VectorIndexWriter adapter 已支持 draft point upsert。
- Indexing runtime 会从 active_config 构造 embedding client、Qdrant writer、collection 和模型参数。
- 已新增并执行通过 Qdrant / embedding provider opt-in 联调测试，覆盖临时 collection 创建、真实 embedding、draft point upsert、payload 激活和带权限 filter 的向量召回。
- draft 写入完成后标记 `index_versions(status=ready)`。
- publish 前会校验 chunk 数量、draft vector refs 数量和 embedding dimension，失败时不会激活索引。
- active index 发布：
  - 先激活 Qdrant point payload。
  - 旧 active index/version 归档。
  - 新 index_version 激活。
  - document_version 激活。
  - document 切换为 `active/indexed`。
  - chunks、chunk_index_refs、keyword_index_entries 切换为 active。

仍缺少：

- 旧索引 pending_delete 清理。
- 删除、权限收紧、索引重建时的 fail closed 闭环。
- Qdrant snapshot / 重建 / 修复的运维入口。
- 发布失败、旧索引清理和权限 payload 刷新的更完整回归测试。

### 3.5 查询、检索、上下文和答案

已完成第一版最小链路：

- `POST /internal/v1/queries` 非流式查询 API。
- 查询请求上下文：request_id、trace_id、active_config_version、permission_version、active_index_version_hash。
- 调用 Permission Service 构建上下文和 filter。
- PostgreSQL `keyword_index_entries` 关键词召回。
- query embedding client，支持 TEI 和 OpenAI-compatible / HTTP provider 的常见响应格式。
- Qdrant VectorRetriever adapter，使用 active index 的 collection_name 和 Permission Service qdrant filter 召回。
- RRF 融合排序，按 chunk 去重并提升多路召回共同命中的候选。
- 候选准入 gate：企业、知识库、文档状态、chunk 状态、active index、visibility_state、权限版本和部门可见性。
- rerank 调用外部 rerank provider，只对权限 gate 后的候选精排；不可用时降级为融合排序。
- citation 返回。
- citation 校验，拒绝无引用或引用未授权 source 的 LLM 答案，并写高风险审计。
- `query_logs` 写入，不保存 query 明文。
- answer 模式已接入 OpenAI-compatible LLM provider；LLM 不可用时以 `degraded=true` 返回检索来源。
- LLM 与 rerank 调用写入 `model_call_logs`，只保存 route、token、耗时和 prompt/input/output hash，不保存 prompt 或文档原文。
- LLM 降级、rerank 降级、citation 校验失败和权限拒绝会写入 `audit_logs`。

仍缺少：

- provider token 级真实流式输出；当前 Query Stream 是服务端缓冲式 SSE 第一版。
- 查询改写与扩展。
- 查询回归数据集验证。
- 更完整的候选归一化和配置化融合参数。

## 6. 当前风险与注意事项

- OpenAPI 已定义的接口多于实际 FastAPI 挂载接口，已通过契约差异测试追踪剩余缺口。
- 数据库表已建，但很多业务服务尚未写入或读取这些表，不能将“有表”视为“功能已完成”。
- Qdrant 写入链路已有单元测试、opt-in 联调测试和真实 Markdown 上传验证；仍需补齐更多失败注入、清理和运维入口。
- Import Job 长时间显示 `queued` 通常表示 Worker 未运行或未消费到任务；`GET /internal/v1/admin/import-jobs` 返回 200 只代表任务列表接口正常，不代表导入成功。
- OCR、复杂版式解析、导入失败补偿、索引清理、文档预览和查询答案生成质量评测仍是 MVP 后续主要工作，应按安全边界和可验证闭环逐段推进。
- 模型调用、查询日志和 citation 校验涉及隐私与可追溯性，不能只依赖 LLM 输出，需要工程校验。
- `make smoke-p0` 仍依赖验收环境提供可登录用户、可访问知识库和可查询文档；当前仓库不保存真实 smoke 密码。

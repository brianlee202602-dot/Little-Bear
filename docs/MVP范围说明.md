# MVP 范围说明

## 1. 文档目标

本文用于冻结 Little Bear RAG 后端第一阶段 MVP 范围，作为后续 OpenAPI、数据库 Schema、权限矩阵、状态机、测试计划和正式编码的输入。

MVP 的目标不是实现完整平台能力，而是跑通一条可上线验证的核心业务闭环：

```text
初始化系统 -> 登录 -> 组织/用户/权限 -> 知识库 -> 文档导入 -> 索引发布 -> 权限过滤查询 -> 带引用答案 -> 审计可追踪
```

## 2. MVP 业务目标

第一阶段系统必须证明以下能力成立：

- 系统可以在只有数据库连接配置的情况下完成首次初始化。
- 业务配置通过初始化接口写入数据库 active config，不依赖环境变量保存业务配置。
- 用户可以登录并获得 access/refresh JWT。
- 管理员可以维护最小组织、用户、角色和权限绑定。
- 管理员或有权限用户可以创建知识库并导入文档。
- Worker 可以异步完成文档解析、切块、向量化和索引写入。
- draft 索引不会被查询命中，只有 active index 可以服务查询。
- 查询必须下推权限过滤，用户只能看到自己有权访问的文档和 chunk。
- 查询答案必须带 citation，citation 可以回溯到文档版本、页码和 chunk。
- 初始化、登录、导入、权限变更、查询和管理员操作必须可审计。

## 3. P0 功能范围

### 3.1 初始化与配置

- [ ] 系统启动时只读取数据库连接配置。
- [ ] 未初始化时只开放 `setup-state`、`setup-config-validations`、`setup-initialization`。
- [ ] setup JWT 每次签发都生成新 token，并使旧 token 失效。
- [ ] 初始化接口创建首个系统管理员、默认企业、默认部门、默认角色和 `active_config v1`。
- [ ] 初始化完成后关闭初始化写接口。
- [ ] ServiceBootstrap 基于 active config 初始化 Redis、Secret Store、对象存储、向量库、关键词检索、模型网关 mock、审计和限流。

### 3.2 认证与会话

- [ ] 本地账号登录。
- [ ] access JWT 签发和校验。
- [ ] refresh JWT 签发、状态保存和 rotation。
- [ ] 当前用户信息查询。
- [ ] 修改当前用户密码。
- [ ] 管理员创建、禁用、软删除用户。
- [ ] 管理员重置密码和解锁用户。

### 3.3 组织与权限

- [ ] 默认企业和默认部门。
- [ ] 部门创建、修改、禁用、删除。
- [ ] 用户所属部门维护。
- [ ] 内置角色初始化。
- [ ] 用户角色绑定、替换和撤销。
- [ ] Permission Service 构建权限上下文。
- [ ] 文档可见性只支持 `department` 和 `enterprise`。
- [ ] 权限过滤条件下推到向量检索和关键词检索。
- [ ] 权限收紧时先写 access block，再刷新索引 payload。

### 3.4 知识库与文档

- [ ] 知识库创建、查询、修改、禁用、删除。
- [ ] 文件夹创建、查询、修改、移动、归档。
- [ ] 文档元数据创建、查询、修改、删除。
- [ ] 文档版本记录。
- [ ] 文档预览。
- [ ] 文档删除先阻断查询，再异步清理索引。

### 3.5 导入与 Worker

- [ ] 上传文件创建导入任务。
- [ ] URL 或批量元数据创建导入任务。
- [ ] 导入任务查询。
- [ ] 导入任务取消。
- [ ] 导入任务重试。
- [ ] Worker 基于数据库任务表领取任务。
- [ ] Worker 阶段化执行：解析、清洗、切块、embedding、索引写入、发布。
- [ ] Worker 崩溃后可通过 `locked_until` 释放任务。

### 3.6 索引

- [ ] 写入 Qdrant draft 向量索引。
- [ ] 写入关键词 draft 索引。
- [ ] 索引 payload 包含权限字段、文档状态、chunk 状态和索引版本信息。
- [ ] 发布前校验 chunk 数量、向量维度、payload hash、权限快照和 draft 不可见。
- [ ] 发布 active index。
- [ ] 旧索引归档或 pending_delete。
- [ ] 删除、权限收紧和索引重建时 fail closed。

### 3.7 查询与答案

- [ ] 非流式查询 API。
- [ ] 流式查询 API，严格引用模式下不得输出未经引用校验的最终 token。
- [ ] 查询请求创建 request context。
- [ ] 构建权限上下文。
- [ ] 查询改写和扩展可先使用规则或模型 mock。
- [ ] 向量召回和关键词召回。
- [ ] 候选片段准入校验：权限、文档状态、chunk 状态、active index、access block、权限快照。
- [ ] rerank 可先使用模型 mock 或简化排序。
- [ ] Context Builder 组装引用上下文。
- [ ] LLM 可先使用 mock，返回可控答案。
- [ ] citation 校验。
- [ ] 查询日志、审计和指标记录。

### 3.8 审计与可观测

- [ ] 结构化日志包含 `request_id`、`trace_id`、`user_id`、`enterprise_id`、`stage`、`latency_ms`、`status`。
- [ ] 审计记录覆盖初始化、登录、用户管理、角色绑定、权限变更、配置发布、文档导入、文档删除、查询和模型调用。
- [ ] 查询日志可按用户、知识库、状态、是否降级、配置版本和权限版本筛选。
- [ ] 模型调用日志记录模型、路由、耗时、token 用量和错误，不记录完整 prompt。
- [ ] `GET /health/live` 和 `GET /health/ready`。

## 4. P0 暂不实现范围

以下能力不进入第一阶段正式编码范围，避免 MVP 失焦：

- [ ] 外部组织源同步，例如 LDAP、飞书、企业微信、钉钉。
- [ ] 多级部门递归权限、上级部门可见、子部门可见。
- [ ] 指定用户共享、项目组共享、自定义组织 ID 共享。
- [ ] 完整审批流和双人复核工作台。
- [ ] 多租户运营后台。
- [ ] 复杂计费、配额和成本分摊。
- [ ] 完整离线评测平台。
- [ ] 多模型供应商生产级动态路由。
- [ ] 真实大模型高可用调度。
- [ ] 复杂 Agent 任务规划、多 Agent 协作。
- [ ] 连接器市场和复杂外部数据源增量同步。
- [ ] 面向终端用户的完整前端页面实现。
- [ ] 复杂报表和 BI 分析。

说明：

- P0 可以保留接口或扩展点，但不得为了暂不实现能力引入复杂状态、复杂权限模型或跨模块耦合。
- 模型相关能力在 P0 允许使用 mock 或单一模型 adapter，只要接口和日志契约稳定。

## 5. MVP 成功标准

### 5.1 功能成功标准

- [ ] 从空数据库完成首次初始化。
- [ ] 初始化完成后普通业务 API 可用，初始化写接口关闭。
- [ ] 管理员可以创建用户、部门、角色绑定和知识库。
- [ ] 用户上传文档后，可以通过导入任务追踪处理进度。
- [ ] Worker 可以把文档处理到 active index。
- [ ] 有权限用户可以查询到文档答案和 citation。
- [ ] 无权限用户查询不到不属于自己部门且非企业公开的文档。
- [ ] 文档删除后，即使索引物理删除失败，也不能被查询命中。
- [ ] 权限收紧后，旧用户不能通过缓存或旧索引访问。
- [ ] 查询答案中的 citation 能回溯到 active chunk、document version、页码和对象存储原文。

### 5.2 安全成功标准

- [ ] 所有业务 API 默认需要 `Authorization: Bearer <jwt>`。
- [ ] setup JWT 不能访问普通业务 API、管理员 API 或服务间 API。
- [ ] JWT 中不保存完整权限上下文。
- [ ] 权限判断不依赖 LLM。
- [ ] 权限过滤不能只在召回后做应用层过滤，必须下推到向量检索和关键词检索。
- [ ] API 响应、普通日志和审计摘要不得泄露 secret、密码、token、完整 prompt 或未脱敏敏感原文。
- [ ] 高风险操作必须记录审计。

### 5.3 工程成功标准

- [ ] 所有 P0 API 都有 OpenAPI 契约。
- [ ] 所有 P0 表都有 migration。
- [ ] 所有 P0 状态机都有允许迁移表。
- [ ] 所有 P0 写接口都有权限矩阵和审计定义。
- [ ] P0 自动化测试覆盖初始化、认证、权限、导入、索引、查询和删除阻断。
- [ ] 本地开发环境不依赖真实大模型即可跑通 P0 流程。

## 6. 第一阶段端到端验收路径

### 6.1 初始化验收

1. 启动 API、Worker 和 Model Gateway 管理进程。
2. 系统只读取数据库连接配置。
3. 调用 `GET /internal/v1/setup-state`，返回未初始化。
4. 使用受控 CLI 签发 setup JWT。
5. 调用 `POST /internal/v1/setup-config-validations` 校验初始化配置。
6. 调用 `PUT /internal/v1/setup-initialization` 创建首个系统管理员、默认企业、默认部门、默认角色和 `active_config v1`。
7. `GET /health/ready` 返回 ready。
8. 再次调用初始化写接口被拒绝。

### 6.2 管理员配置验收

1. 系统管理员登录。
2. 创建销售部门。
3. 创建普通用户 Alice。
4. 将 Alice 加入销售部门。
5. 为 Alice 绑定普通员工角色。
6. 创建销售知识库，默认可见性为 `department`。
7. 创建或确认审计日志记录上述操作。

### 6.3 文档导入验收

1. 管理员或有权限用户上传销售政策文档。
2. API 创建 document、document_version 和 import_job。
3. Worker 领取任务。
4. Worker 完成解析、清洗、切块、embedding 和 draft 索引写入。
5. 发布前校验 draft 索引不可见。
6. 发布 active index。
7. import_job 状态变为 success。
8. 文档状态变为 active/indexed。

### 6.4 权限查询验收

1. Alice 登录。
2. Alice 查询销售政策问题。
3. 查询链路生成权限过滤条件。
4. 向量召回和关键词召回都携带权限过滤。
5. 候选片段通过准入校验。
6. 返回答案和 citation。
7. citation 可回溯到文档版本、页码和 chunk。

### 6.5 越权阻断验收

1. 创建非销售部门用户 Bob。
2. Bob 查询销售政策问题。
3. Bob 不应召回销售部门可见文档。
4. 将销售政策文档可见性改为 `enterprise`。
5. Bob 可以查询到该文档。
6. 将可见性改回 `department`。
7. Bob 立即无法查询到该文档，旧缓存不可复用。
8. 审计记录权限扩大和权限收紧。

### 6.6 删除阻断验收

1. 管理员删除销售政策文档。
2. 系统先写 access block。
3. 即使 Qdrant 或关键词索引物理删除失败，Alice 也不能查询到该文档。
4. 物理删除失败进入重试队列并告警。
5. 审计记录删除操作和删除任务状态。

## 7. MVP 与后续文档的关系

- OpenAPI 契约必须覆盖本文 P0 API。
- 数据库 Schema 必须支持本文 P0 数据闭环。
- 权限矩阵必须覆盖本文 P0 写接口和查询接口。
- 状态机设计必须覆盖本文 P0 初始化、导入、文档、索引、配置和 token 生命周期。
- 测试计划必须覆盖本文端到端验收路径。

## 8. 变更规则

MVP 范围冻结后，如需新增 P0 功能，必须同时说明：

- 新增功能解决的业务问题。
- 不做该功能会阻断哪条 P0 验收路径。
- 需要新增或修改的 API。
- 需要新增或修改的数据表。
- 需要新增或修改的权限矩阵。
- 需要新增或修改的状态机。
- 需要新增或修改的测试用例。

没有影响 P0 验收路径的功能，应进入 P1 或后续版本。

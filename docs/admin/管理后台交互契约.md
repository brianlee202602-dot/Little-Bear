# 管理后台交互契约

## 1. 文档目标

本文定义 Little Bear RAG 管理后台第一批交互契约，作为前端路由、菜单权限、列表筛选、表单校验、高风险确认、审计详情展示和联调验收的共同输入。

本文只定义管理后台交互边界，不新增后端 P0 功能范围。接口、scope、审计和确认要求以 `docs/contracts/openapi.yaml`、`docs/contracts/权限矩阵.md` 和 `docs/contracts/审计事件字典.md` 为准。

## 2. 设计边界

- 管理后台使用登录用户的 access JWT。
- 菜单隐藏不是安全边界；所有管理 API 必须由后端校验 token、scope 和资源范围。
- 管理后台不得直接调用模型 Provider Adapter service API。
- P1 交互契约可以覆盖 P0 后端已有管理接口，但不得要求外部组织同步、配置回滚、配置 diff、指标汇总、告警中心、审计导出或完整审批流。
- 所有敏感字段展示必须服从审计事件字典和脱敏规则。

## 4. 菜单与 Scope 映射

菜单可见性使用 `GET /internal/v1/sessions/current` 返回的 scopes 摘要，但前端缓存只用于展示控制。

| 一级菜单 | 二级菜单 | 可见条件 | 主要能力 |
| --- | --- | --- | --- |
| 任务中心 | 导入任务 | `import_job:read` | 查看导入任务与失败摘要 |
| 知识库管理 | 知识库 | `knowledge_base:manage` | 知识库增删改查 |
| 知识库管理 | 文件夹 | `folder:manage` | 文件夹创建、移动、归档、删除 |
| 知识库管理 | 文档 | `document:manage` | 文档元数据、状态、删除 |
| 知识库管理 | 索引 | `document:index` | 查看索引版本、创建重建任务 |
| 组织账号 | 用户 | `user:read` 或 `user:manage` | 用户查询、创建、禁用、删除、重置密码 |
| 组织账号 | 部门 | `org:read` 或 `org:manage` | 部门查询、创建、禁用、删除 |
| 权限中心 | 角色 | `role:read` 或 `role:manage` | 角色查询、创建、修改、删除 |
| 权限中心 | 角色绑定 | `role:read` 或 `role:manage` | 用户角色绑定维护 |
| 权限中心 | 权限策略 | `permission:evaluate` 或 `permission:manage` | 权限评估、资源权限修改 |
| 配置中心 | 配置项 | `config:read` 或 `config:manage` | 配置查看、草稿保存、校验、发布 |
| 审计中心 | 审计日志 | `audit:read` | 审计、查询日志、模型调用日志 |
| 系统状态 | 健康检查 | `health:read` 或任一管理 scope | 查看 ready/live 摘要 |

操作按钮可见性：

- 创建、编辑、删除类按钮必须按对应 `*:manage` scope 判断。
- 只读 scope 只能展示列表、详情和只读字段。
- 高风险按钮即使可见，也必须在提交前执行二次确认。
- 后端返回 `403`、`SETUP_REQUIRED`、`SETUP_CLOSED` 或资源范围拒绝时，前端必须展示结构化错误，不得本地猜测权限。

## 5. 表格筛选字段

`page` 和 `page_size` 是所有分页列表的通用字段。默认 `page=1`，`page_size=20`，最大 `page_size=200`。

| 页面 | OpenAPI 当前字段 | 建议后续字段 | 默认排序 |
| --- | --- | --- | --- |
| 导入任务 | `page`、`page_size`、`status`、`stage`、`kb_id`、`created_by` | `created_after`、`created_before` | 创建时间倒序 |
| 知识库 | `page`、`page_size`、`keyword`、`status` | `owner_department_id`、`default_visibility` | 更新时间倒序 |
| 文件夹 | path `kb_id` | `keyword`、`status`、`parent_id` | 名称升序 |
| 文档 | path `kb_id`、`page`、`page_size`、`status` | `keyword`、`folder_id`、`visibility`、`index_status`、`owner_department_id` | 更新时间倒序 |
| 用户 | `page`、`page_size`、`keyword`、`status` | `department_id`、`role_id` | 创建时间倒序 |
| 部门 | `page`、`page_size`、`keyword`、`status` | 无 | code 升序 |
| 角色 | 无分页参数 | `keyword`、`status`、`scope_type`、`is_builtin` | code 升序 |
| 配置项 | `page`、`page_size` | `key`、`status`、`scope_type` | key 升序 |
| 配置版本 | 无分页参数 | `status`、`risk_level`、`created_by` | 版本倒序 |
| 审计日志 | `page`、`page_size`、`actor_id`、`action`、`resource_type`、`result`、`risk_level` | `resource_id`、`created_after`、`created_before` | 创建时间倒序 |
| 查询日志 | `page`、`page_size`、`user_id`、`kb_id`、`degraded` | `status`、`config_version`、`permission_version`、`created_after`、`created_before` | 创建时间倒序 |
| 模型调用日志 | `page`、`page_size`、`model`、`degraded` | `error_code`、`config_version`、`created_after`、`created_before` | 创建时间倒序 |

前端约束：

- `建议后续字段` 不得在 OpenAPI 支持前作为必填联调条件。
- 过滤条件必须反映在 URL query 中，便于刷新和排障复现。
- 列表请求失败时必须展示 `request_id` 和 `error_code`。
- 清空筛选必须恢复 OpenAPI 默认分页值。

## 7. 高风险确认机制

所有 `x-confirmation-required=true` 或 `conditional` 命中的操作必须弹出二次确认。确认请求不得只依赖前端文案；提交时必须携带后端定义的确认字段或幂等请求标识，具体字段以后端实现为准。

| 操作 | 触发条件 | 接口 | 确认类型 | 确认文案要点 |
| --- | --- | --- | --- | --- |
| 创建企业可见知识库 | `default_visibility=enterprise` | `POST /internal/v1/admin/knowledge-bases` | 条件确认 | 确认该知识库默认对全企业可见 |
| 扩大知识库可见性 | visibility expands | `PATCH /internal/v1/admin/knowledge-bases/{kb_id}` | 条件确认 | 确认会扩大可访问范围并刷新缓存 |
| 删除知识库 | always | `DELETE /internal/v1/admin/knowledge-bases/{kb_id}` | 强确认 | 输入知识库名称；说明会先阻断查询再异步清理 |
| 删除文件夹 | always | `DELETE /internal/v1/admin/folders/{folder_id}` | 强确认 | 输入文件夹名称；说明子资源查询会被阻断 |
| 扩大文档可见性 | visibility expands | `PATCH /internal/v1/admin/documents/{doc_id}` | 条件确认 | 确认文档可能被更多用户召回 |
| 删除文档 | always | `DELETE /internal/v1/admin/documents/{doc_id}` | 强确认 | 输入文档标题；说明 access block 立即生效 |
| 重建索引 | always | `POST /internal/v1/admin/documents/{doc_id}/index-jobs` | 强确认 | 确认会创建异步任务，旧 active index 继续服务 |
| 创建管理员用户 | granting admin role | `POST /internal/v1/admin/users` | 条件确认 | 确认授予管理权限 |
| 禁用管理员 | disabling admin | `PATCH /internal/v1/admin/users/{user_id}` | 条件确认 | 确认会撤销旧会话并失效权限缓存 |
| 删除用户 | always | `DELETE /internal/v1/admin/users/{user_id}` | 强确认 | 输入用户名；说明账号将软删除 |
| 重置密码 | always | `PUT /internal/v1/admin/users/{user_id}/password` | 强确认 | 确认不会展示或记录明文密码 |
| 删除部门 | always | `DELETE /internal/v1/admin/departments/{department_id}` | 强确认 | 输入部门 code；说明会影响组织版本和权限缓存 |
| 移除主部门 | removing primary dept | `PUT /internal/v1/admin/users/{user_id}/departments` | 条件确认 | 确认用户权限上下文会变化 |
| 创建/修改高风险角色 | high-risk scopes | `POST/PATCH /internal/v1/admin/roles...` | 条件确认 | 确认包含高风险 scope |
| 删除角色 | always | `DELETE /internal/v1/admin/roles/{role_id}` | 强确认 | 输入角色 code；内置角色不可删除 |
| 新增高风险角色绑定 | high-risk role | `POST /internal/v1/admin/users/{user_id}/role-bindings` | 条件确认 | 确认授予高风险角色 |
| 替换用户角色绑定 | always | `PUT /internal/v1/admin/users/{user_id}/role-bindings` | 强确认 | 输入用户名；说明会整体替换角色 |
| 移除最后管理员 | removing last admin | `DELETE /internal/v1/admin/users/{user_id}/role-bindings/{binding_id}` | 条件确认或后端拒绝 | 优先后端拒绝；若允许必须强确认 |
| 修改资源权限 | always | `PUT /internal/v1/knowledge-bases/{kb_id}/permissions`、`PUT /internal/v1/documents/{doc_id}/permissions` | 强确认 | 确认可见性变化、权限版本和索引刷新 |
| 保存高风险配置 | high-risk key | `PUT /internal/v1/admin/configs/{key}` | 条件确认 | 确认配置会影响运行链路 |
| 发布配置版本 | always | `PATCH /internal/v1/admin/config-versions/{version}` | 强确认 | 输入版本号；说明失败时 active 指针不变 |

确认弹窗必须展示：

- 操作对象名称和 ID。
- 当前值和目标值摘要。
- 影响范围，例如用户数、知识库、文档、索引或配置版本。
- 后端要求的 `risk_level` 和补偿策略摘要。
- 提交后返回的 `request_id`。

## 8. 审计详情展示脱敏规则

审计中心只能读取受控审计表、查询日志和模型调用日志，不得直接读取普通应用日志。

| 数据类型 | 可展示 | 禁止展示 |
| --- | --- | --- |
| token | `jti`、`token_type`、hash、过期时间 | access / refresh / setup / service token 明文 |
| secret | `secret_ref`、provider、路径摘要 | secret value、API key、对象存储 access key |
| 密码 | 操作类型、策略校验结果 | 明文密码、密码摘要材料、重置密码明文 |
| query | `query_hash`、长度、状态、降级原因 | 未脱敏用户输入全文 |
| prompt | `prompt_template_id`、版本、hash、变量摘要 | 完整 prompt |
| 文档内容 | `content_hash`、页码、chunk_id、限长 `text_preview` | 未脱敏文档原文、大段 chunk 正文 |
| 个人信息 | 用户 ID、部门、角色、掩码邮箱/手机号 | 完整个人敏感信息 |
| 模型调用 | `model_type`、`model_name`、`model_route_hash`、`latency_ms`、`token_usage_json`、`error_code` | vendor token、完整输入输出 |

审计详情页字段：

- 基础信息：`id`、`event_name`、`created_at`、`actor_type`、`actor_id`、`action`、`result`、`risk_level`、`error_code`。
- 资源信息：`resource_type`、`resource_id`、`config_version`、`permission_version`、`index_version_hash`。
- 摘要信息：脱敏后的 `summary_json`。
- 排障信息：`request_id`、`trace_id`、`error_code`、`retryable`。

展示规则：

- 后端必须先对 `summary_json` 执行脱敏和字段裁剪；前端只展示后端返回内容。
- `summary_json` 中出现已脱敏字段时，前端显示为 `[redacted]`，并保留字段名用于排查。
- 查询日志详情不得展示完整 query；只展示 hash、长度、知识库、状态、是否降级和版本信息。
- 模型调用日志不得展示完整 prompt 或完整模型输出。
- chunk 详情只展示 `text_preview`，不得提供原文下载入口；若后续支持下载，必须使用短期签名 URL 并重新做权限校验和审计。

## 9. 验收标准

- 每个管理页面都能映射到一个或多个 OpenAPI endpoint。
- 每个菜单入口都有明确 scope，可由当前用户 scopes 摘要控制显示。
- 每个列表页都有 OpenAPI 已支持的筛选字段，并标明未支持的后续字段。
- 每个写表单都能映射到 request schema，并声明前端基础校验和后端最终校验边界。
- 所有高风险操作都有确认触发条件、确认类型和影响说明。
- 审计详情页不会展示密码、token、secret value、完整 prompt、未脱敏用户输入或大段文档原文。

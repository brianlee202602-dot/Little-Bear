# 模型与检索评测基线

## 1. 文档目标

本文定义 Little Bear RAG 后端 P1 阶段的模型与检索评测基线，作为检索调参、模型路由变更、配置发布评审、上线前质量回归和故障排查的共同输入。

本文不是完整离线评测平台设计，也不把复杂评测工作台或自动化标注系统纳入 P0 编码门禁。P0 以外部 embedding、rerank、LLM provider 跑通初始化、导入、索引、权限过滤查询、citation 和审计闭环为准。

## 3. 范围边界

P1 要解决：

- 定义最小离线评测集格式和样例。
- 定义检索 `recall@k`、citation 命中率、无答案拒答率、延迟和成本指标。
- 定义评测运行模式、报告格式、失败分级和变更触发条件。
- 确保评测能发现权限越权、draft index 暴露、缓存串权和 citation 失效。
- 为后续模型、检索、chunk、rerank 和配置变更提供回归基线。

P1 不解决：

- 不建设完整评测平台、在线 A/B 平台或人工标注工作台。
- 不依赖 LLM judge 作为唯一质量判定。
- 不保存生产用户原始 query、完整 prompt、文档原文或模型输出全文。
- 不允许为了提升指标绕过权限过滤、citation 校验、access block 或脱敏规则。
- 不把真实外部模型供应商接入作为 P0 或 3.4 完成条件。

## 4. 设计原则

- 安全指标优先于质量指标：权限泄露、draft index 暴露、删除文档命中、缓存串权必须为 0。
- 可复现优先：评测必须记录 dataset 版本、代码版本、config version、index version、model route hash 和 prompt/template 版本。
- 规则优先，模型辅助：检索命中、citation 有效性、权限泄露和无答案拒答用规则判断；LLM judge 只能作为人工复核辅助。
- 分层评测：检索、rerank、context、answer、citation、延迟、成本分别统计，避免只看最终答案。
- 不污染审计：评测运行可以写 `query_logs` 和 `model_call_logs`，但不得写入未脱敏 query 原文或 prompt。
- 阈值先保守：首版阈值用于发现明显退化，后续根据真实业务样本迭代。

## 5. 最小离线评测集

### 5.1 数据集文件

建议从以下本地样例开始：

- `docs/examples/eval-dataset.local.p1.jsonl`

该文件是 P1 评测集样例，不是 P0 测试夹具。后续正式评测集可以按业务域拆分，例如：

- `eval-sales-policy.v1.jsonl`
- `eval-hr-policy.v1.jsonl`
- `eval-it-support.v1.jsonl`

### 5.2 JSONL 字段

每行代表一个评测用例。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `eval_id` | string | 是 | 稳定用例 ID |
| `dataset_version` | string | 是 | 数据集版本 |
| `query` | string | 是 | 离线评测 query，只允许使用脱敏或合成问题 |
| `locale` | string | 是 | 例如 `zh-CN` |
| `actor_fixture` | string | 是 | `alice_sales`、`bob_non_sales`、`admin` 等 |
| `kb_ids` | string[] | 是 | 评测知识库 fixture ID |
| `answer_type` | string | 是 | `answerable`、`no_answer`、`permission_denied` |
| `expected_doc_ids` | string[] | 否 | 期望命中文档 ID |
| `expected_chunk_ids` | string[] | 否 | 期望命中 chunk ID，可在导入后由 fixture resolver 映射 |
| `expected_citation_refs` | object[] | 否 | 期望 citation，包含 doc、document_version、page 范围或 chunk |
| `forbidden_doc_ids` | string[] | 否 | 禁止命中文档 ID |
| `must_include` | string[] | 否 | 最终答案应包含的事实短语 |
| `must_not_include` | string[] | 否 | 最终答案不得包含的内容 |
| `tags` | string[] | 是 | 业务、难度和风险标签 |
| `notes` | string | 否 | 标注说明 |

`query` 字段只允许进入离线评测文件和评测 runner 内存，不得写入 `audit_logs.summary_json`、普通日志或默认查询日志。评测报告中使用 `query_hash`。

### 5.3 最小集规模

P1 初始最小集建议 30 条：

| 类型 | 数量 | 目的 |
| --- | --- | --- |
| 单文档可回答 | 10 | 验证基础召回和 citation |
| 多文档可回答 | 5 | 验证融合、rerank 和上下文组合 |
| 表格、条款或页码敏感 | 5 | 验证 chunk、页码和 citation 精度 |
| 无答案 | 5 | 验证拒答和不编造 |
| 权限阻断 | 5 | 验证 Bob、旧权限、删除和 draft index 不泄露 |

本地样例可少于 30 条，用于 runner 和报告格式联调；进入 release 基线前必须补齐 30 条。

## 6. 评测准备

评测运行前必须固定以下上下文：

| 上下文 | 要求 |
| --- | --- |
| 数据库 | 使用独立 `eval_run_id` 或测试企业，避免污染开发数据 |
| 文档导入 | 评测文档必须完成 active index 发布 |
| 权限 | 至少包含 Alice 销售部门用户和 Bob 非销售部门用户 |
| 配置 | 记录 `config_version` 和检索、chunk、model、timeout 关键配置 |
| 索引 | 记录 active `index_version_hash` 和 embedding 维度 |
| 模型 | 使用外部 embedding、rerank、LLM provider；记录 `model_route_hash` |
| 缓存 | 默认冷缓存运行一次，热缓存可作为单独 profile |

评测 runner 不应直接访问 Qdrant 或关键词索引绕过业务链路。最终质量回归必须通过 `POST /internal/v1/queries` 或内部 Query Service 等价入口执行，确保权限、缓存、citation 和审计路径都被覆盖。

## 7. 指标定义

### 7.1 检索 Recall@K

适用样本：`answer_type=answerable`。

定义：

```text
doc_recall@k = top_k 候选中包含任一 expected_doc_ids 的样本数 / answerable 样本数
chunk_recall@k = top_k 候选中包含任一 expected_chunk_ids 的样本数 / answerable 且有 chunk 标注的样本数
strict_recall@k = top_k 候选中包含全部 required expected_doc_ids 的样本数 / 多文档 answerable 样本数
```

首版统计 `k=5`、`k=10`、`k=20`。如果 API 只返回最终 citation，不返回候选列表，则先用 citation 近似估计 `doc_recall`，同时在 Query Service 内部报告中输出候选摘要。

初始阈值：

| 指标 | local-provider 最低值 | release 建议最低值 |
| --- | --- | --- |
| `doc_recall@5` | 0.80 | 0.85 |
| `doc_recall@10` | 0.90 | 0.92 |
| `chunk_recall@10` | 0.75 | 0.80 |
| `strict_recall@20` | 0.70 | 0.75 |

说明：阈值是第一版保护线，不代表生产质量承诺；首批真实业务样本补齐后应重新校准。

### 7.2 Citation 命中率

适用样本：`answer_type=answerable`。

定义：

```text
citation_valid_rate = 所有返回 citation 都能回溯到 active chunk/document_version/page 且用户有权限的样本数 / 返回 citation 的样本数
citation_hit_rate = 至少一个 citation 命中 expected_citation_refs 或 expected_doc_ids 的样本数 / answerable 样本数
citation_precision = 命中 expected/acceptable citation 的返回 citation 数 / 返回 citation 总数
```

硬性要求：

- `citation_valid_rate = 1.0`
- citation 不得引用 draft、deleted、blocked、archived 或无权限文档。
- strict citation 模式下，citation 校验失败不得输出最终答案。

初始阈值：

| 指标 | 最低值 |
| --- | --- |
| `citation_valid_rate` | 1.00 |
| `citation_hit_rate` | 0.85 |
| `citation_precision` | 0.80 |

### 7.3 无答案拒答率

适用样本：`answer_type=no_answer`。

定义：

```text
no_answer_refusal_rate = 返回明确无依据/无法回答/需要更多信息且不输出编造事实的样本数 / no_answer 样本数
unsupported_answer_rate = 输出未被 citation 支撑的业务事实样本数 / no_answer 样本数
```

要求：

- `no_answer_refusal_rate >= 0.95`
- `unsupported_answer_rate = 0`
- no_answer 样本可以返回检索为空、低置信度或结构化拒答，但不得为了凑答案引用无关文档。

### 7.4 权限安全指标

适用样本：所有样本，尤其是 `answer_type=permission_denied` 和权限风险标签。

硬性要求：

| 指标 | 要求 |
| --- | --- |
| `permission_violation_count` | 0 |
| `forbidden_doc_hit_count` | 0 |
| `draft_index_exposure_count` | 0 |
| `deleted_or_blocked_doc_hit_count` | 0 |
| `cache_permission_leak_count` | 0 |

任一安全指标非 0，评测结果直接失败，不允许用更高 recall 或更低延迟抵消。

### 7.5 答案事实覆盖率

适用样本：`answer_type=answerable` 且有 `must_include` 标注。

定义：

```text
must_include_hit_rate = 命中全部 must_include 短语或等价事实的样本数 / 有 must_include 标注的 answerable 样本数
must_not_include_violation_rate = 出现 must_not_include 内容的样本数 / 所有样本数
```

首版可以用规则短语匹配；同义表达允许人工复核，不建议用 LLM judge 自动放行。

初始阈值：

- `must_include_hit_rate >= 0.80`
- `must_not_include_violation_rate = 0`

### 7.6 延迟指标

延迟按端到端和阶段分别统计。

| 指标 | 说明 | 初始目标 |
| --- | --- | --- |
| `query_total_p50_ms` | 查询总耗时 P50 | <= 1500ms local-provider |
| `query_total_p95_ms` | 查询总耗时 P95 | <= active config `timeout.query_total_ms` |
| `retrieval_p95_ms` | 召回和融合 P95 | <= 1500ms |
| `rerank_p95_ms` | rerank P95 | <= active config `timeout.rerank_ms` |
| `llm_first_token_p95_ms` | 流式首 token P95 | adapter profile 单独统计 |
| `citation_verify_p95_ms` | citation 校验 P95 | <= 500ms |

不同 provider profile 的延迟不得混跑，必须单独标注 embedding、rerank 和 LLM 服务版本。

### 7.7 成本指标

成本指标不要求 P0 实现计费系统，但模型调用日志必须提供可计算摘要。

| 指标 | 来源 |
| --- | --- |
| `embedding_calls_per_query` | `model_call_logs` |
| `rerank_calls_per_query` | `model_call_logs` |
| `llm_calls_per_query` | `model_call_logs` |
| `input_tokens_per_query` | `token_usage_json` |
| `output_tokens_per_query` | `token_usage_json` |
| `estimated_cost_per_1000_queries` | 模型单价配置或离线参数 |

初始要求：

- local-provider profile 可按本地资源成本记为 0，但仍记录调用次数和耗时。
- external-provider profile 必须输出成本估算，不达标时标记 `cost_regression`。
- 成本上升超过 20% 且质量无明显提升时，配置变更不得自动通过。

## 8. 运行模式

| 模式 | 数据集 | 触发时机 | 是否阻断 |
| --- | --- | --- | --- |
| `eval-smoke` | 5 到 10 条样例 | 本地开发、runner 联调 | 不阻断 |
| `eval-nightly` | >= 30 条基线 | nightly 或合并后 | 安全指标阻断，质量指标告警 |
| `eval-release` | 当前 release 基线 | 发布前、检索/模型配置变更前 | 阻断高风险退化 |
| `eval-config-change` | 受影响业务集 | `retrieval`、`chunk`、`model`、`keyword_search` 配置变更 | 阻断安全指标和显著质量退化 |
| `eval-provider-change` | 脱敏业务集 | 接入或切换外部模型 provider 时 | 人工评审，不作为 P0 门禁 |

显著质量退化建议定义：

- `doc_recall@10` 下降超过 5 个百分点。
- `citation_hit_rate` 下降超过 5 个百分点。
- `no_answer_refusal_rate` 下降超过 3 个百分点。
- `query_total_p95_ms` 上升超过 30%。
- 成本估算上升超过 20%。

## 9. 评测报告

评测报告建议输出 JSON，同时生成简短 Markdown 摘要。

### 9.1 报告字段

| 字段 | 说明 |
| --- | --- |
| `eval_run_id` | 本次评测 ID |
| `dataset_name` / `dataset_version` | 数据集标识 |
| `started_at` / `finished_at` | 时间 |
| `code_version` | commit hash 或构建版本 |
| `config_version` | active config version |
| `index_version_hash` | active index 集合摘要 |
| `model_route_hash` | 模型路由摘要 |
| `profile` | local-provider/external-provider/release |
| `metrics` | 指标结果 |
| `thresholds` | 本次阈值 |
| `failed_cases` | 失败样本摘要，只含 `eval_id`、`query_hash` 和错误原因 |
| `security_findings` | 权限、draft、删除阻断、缓存串权发现 |
| `artifact_refs` | 报告文件路径或对象存储引用 |

### 9.2 失败样本摘要

失败样本不得保存完整 query、prompt、文档正文或模型输出全文。推荐字段：

- `eval_id`
- `query_hash`
- `actor_fixture`
- `answer_type`
- `failure_type`
- `expected_doc_ids`
- `returned_doc_ids`
- `returned_citation_refs`
- `request_id`
- `trace_id`
- `error_code`

## 10. 失败分级

| 级别 | 条件 | 处理 |
| --- | --- | --- |
| P0 safety blocker | 权限泄露、draft/deleted/blocked 文档命中、缓存串权、citation 无效仍输出答案 | 阻断发布或配置变更，必须修复 |
| P1 quality blocker | release 模式下 recall/citation/refusal 低于阈值或显著退化 | 阻断相关检索/模型配置变更 |
| P1 performance blocker | P95 超过 active config timeout 或错误率明显升高 | 阻断发布，定位慢阶段 |
| warning | 成本上升、个别样本失败、质量轻微下降 | 记录风险，人工评审是否接受 |

## 11. 变更触发规则

以下变更必须至少运行 `eval-config-change`：

- `/retrieval/*`
- `/chunk/*`
- `/keyword_search/*`
- `/model/rerank_model`
- `/model/llm_model`
- `/model_gateway/routes`
- prompt/template 版本变更
- rerank 策略或 fusion 参数变更

以下变更必须额外人工评审：

- `/model/embedding_model`
- `/model/embedding_version`
- `/model/embedding_dimension`
- `/model/embedding_normalize`
- 向量库 distance、collection 或 payload schema 变更

embedding 语义空间或维度变化不能只靠评测放行，还必须满足索引兼容检查和重建计划。

## 12. 数据安全

- 评测集默认使用合成或脱敏数据。
- 禁止把生产 query 原文直接落入评测集。
- 如果业务方提供真实样本，必须先完成脱敏、授权、用途标注和保留期限确认。
- 评测报告只保存 hash、ID、指标和脱敏摘要。
- 评测 runner 日志不得打印完整 query、prompt、文档原文、token、secret value 或 API key。
- 评测文件应按代码或受控对象存储权限管理，不放在公开 bucket。

## 13. Runner 接口建议

P1 最小 runner 可以是 CLI，不必新增后端 API。

建议命令：

```bash
rag-admin eval run \
  --dataset docs/examples/eval-dataset.local.p1.jsonl \
  --profile local-provider \
  --base-url http://localhost:8000 \
  --output reports/eval/latest.json
```

建议行为：

1. 校验 JSONL schema。
2. 登录或读取受控测试账号 token。
3. 确认 active config、ready、index version 和模型健康。
4. 按 `actor_fixture` 切换用户上下文执行查询。
5. 记录 `request_id`、`trace_id`、query response、query log 和 model call log 摘要。
6. 计算指标并输出报告。
7. 对安全指标非 0 直接返回非零退出码。

## 14. 测试要求

P1 自动化测试至少覆盖：

- JSONL 数据集 schema 校验。
- answerable、no_answer、permission_denied 三类样本解析。
- recall@k、citation hit、refusal、latency、cost 指标计算。
- forbidden document 命中时评测失败。
- citation 指向 draft/deleted/blocked 文档时评测失败。
- 报告中不包含 query 原文、prompt、文档正文、token 或 secret value。
- 相同输入、相同 provider profile 和相同 index 版本下指标可复现。

手工验收至少覆盖：

1. 使用本地 provider 环境导入销售政策 fixture。
2. 运行 `eval-smoke`，确认输出报告包含 config version、index version 和 model route hash。
3. 构造 Bob 查询销售文档样本，确认 forbidden doc hit 为 0。
4. 构造 no_answer 样本，确认返回结构化拒答且无无关 citation。
5. 人工抽查 3 条 answerable 结果，确认答案事实由 citation 支撑。

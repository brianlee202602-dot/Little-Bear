# Org Service 设计实现文档

## 1. 模块目标

Org Service 负责维护企业组织结构、部门层级、项目组、用户组织关系和临时授权组，为权限上下文构建提供稳定的数据来源。

## 2. 模块职责

- 企业、部门、团队、岗位基础数据管理。
- 用户部门关系管理。
- 项目组和临时授权组管理。
- 组织版本号维护。
- 组织结构同步任务。
- 为 Permission Service 提供 subject 集合。

## 3. 非职责

- 不决定文档是否可访问。
- 不直接写向量索引权限字段。
- 不处理登录和会话。
- 不负责知识库权限策略。

## 5. 核心实体

```text
Enterprise
  -> Department
    -> Sub Department
      -> Team

User
  -> Department memberships
  -> Project groups
  -> Temporary groups
```

`departments`：

| 字段 | 说明 |
| --- | --- |
| id | 部门 ID |
| enterprise_id | 企业 ID |
| parent_id | 父部门 |
| name | 部门名称 |
| path | 层级路径 |
| status | active、disabled |
| version | 组织版本 |

`user_department_memberships`：

| 字段 | 说明 |
| --- | --- |
| user_id | 用户 ID |
| department_id | 部门 ID |
| relation_type | primary、secondary、temporary |
| valid_from / valid_to | 有效期 |

`groups`：

| 字段 | 说明 |
| --- | --- |
| id | 组 ID |
| group_type | project、temporary、role_group |
| name | 名称 |
| owner_department_id | 归属部门 |
| valid_from / valid_to | 有效期 |

## 6. 组织同步流程

最小阶段可由管理员手动维护。后续接入外部源时使用同步任务：

```text
1. scheduled-worker 拉取外部组织数据
2. 转换为内部标准结构
3. 校验循环引用、重复部门、无效用户
4. 写入 staging 表
5. 生成 diff
6. 管理员确认或自动发布
7. 更新 departments、memberships、groups
8. 增加 org_version
9. 触发权限缓存失效
```

`UserSubjects` 示例：

```json
{
  "user_id": "user_123",
  "departments": ["dept_rnd_1", "dept_arch"],
  "primary_department": "dept_rnd_1",
  "groups": ["project_alpha", "temp_ipo_review"],
  "org_version": 17

```

## 8. 缓存策略

- 用户 subject 集合可缓存到 Redis。
- 缓存 key 包含 `user_id` 和 `org_version`。
- 组织版本更新后失效相关用户缓存。
- 临时组到期后必须由定时任务清理或在查询时过滤。

## 9. 管理 API

```http
GET  /internal/v1/admin/departments
POST /internal/v1/admin/departments
PATCH /internal/v1/admin/departments/{department_id}

GET  /internal/v1/admin/groups
POST /internal/v1/admin/groups
PATCH /internal/v1/admin/groups/{group_id}

PUT  /internal/v1/admin/users/{user_id}/departments
PUT  /internal/v1/admin/users/{user_id}/groups
```

所有写接口必须校验管理员权限并记录审计。

## 10. 错误码

| 错误码 | 场景 |
| --- | --- |
| `ORG_DEPARTMENT_NOT_FOUND` | 部门不存在 |
| `ORG_GROUP_NOT_FOUND` | 组不存在 |
| `ORG_CYCLE_DETECTED` | 部门层级出现循环 |
| `ORG_MEMBERSHIP_INVALID` | 用户部门关系无效 |
| `ORG_SYNC_FAILED` | 外部组织同步失败 |

## 11. 审计事件

- `org.department.created`
- `org.department.updated`
- `org.department.disabled`
- `org.group.created`
- `org.group.updated`
- `org.membership.changed`
- `org.sync.published`

## 12. 测试点

- 一个用户可属于多个部门。
- 临时组过期后不再进入权限上下文。
- 部门树不能出现循环。
- 组织版本变更会失效权限缓存。
- 普通用户不能修改组织结构。

# Indexing Service 设计实现文档

## 1. 模块目标

Indexing Service 负责向量索引、关键词索引、索引版本、权限 payload、发布和回滚，保证查询链路能在检索层完成权限过滤。

## 2. 模块职责

- 写入 Qdrant 向量索引。
- 写入 PostgreSQL Full Text 关键词索引。
- 维护索引版本。
- 写入和刷新权限字段。
- 校验索引数量、维度、权限 payload。
- 支持文档删除、重建、回滚。
- 为后续迁移 OpenSearch、Milvus 预留接口。

## 3. 非职责

- 不负责文档解析和切块。
- 不负责业务权限计算，只接收 Permission Service 输出的权限快照。
- 不直接暴露给 API route 使用，必须由 Import、Permission 或 Query 模块通过接口调用。

## 8. 索引写入流程

```text
1. 接收 chunks、embeddings、permission_snapshot
2. 创建 index_version(status=draft)
3. 生成确定性 vector_id 和 keyword_id
4. 批量写入 Qdrant，payload.visibility_state=draft
5. 批量写入 PostgreSQL Full Text，visibility_state=draft
6. 写入 chunk_index_refs，visibility_state=draft
7. 校验写入数量、维度、权限 payload 和 payload_hash
8. 使用业务查询过滤条件抽样验证 draft 索引不可见
9. 标记 index_version status=ready
```

写入约束：

- draft/ready 索引不得被查询命中。
- Qdrant 和关键词索引的 payload 必须包含 `enterprise_id`、`kb_id`、`doc_id`、`chunk_id`、`index_version_id`、`visibility_state`、`document_status`、`document_index_status`、`chunk_status`、`permission_version` 和 `is_deleted`。
- `chunk_index_refs` 是索引事实账本，Qdrant/关键词索引写入失败后必须能根据该表补偿或清理。
- active collection 方案和 staging collection 方案都可以，但查询过滤必须只命中 active 索引版本。

## 9. 索引发布

```text
1. Import Worker 完成全部索引写入
2. Indexing Service 校验 chunk 数、向量维度、权限字段
3. PostgreSQL 事务内将 index_version.status 从 ready 改为 active
4. document.current_version_id 指向新 document_version
5. document.index_status = indexed
6. chunk.status = active
7. chunk_index_refs.visibility_state = active
8. 旧版本标记 archived 或 pending_delete
9. 提交后发布配置/索引变更事件，失效查询缓存
10. 写审计事件
```

对于高密级资源，发布前必须确认权限 payload 不为空且符合 strict permission mode。

查询过滤必须包含：

```text
index_version_id IN active_index_versions
AND visibility_state = 'active'
AND document_status = 'active'
AND document_index_status = 'indexed'
AND chunk_status = 'active'
AND is_deleted = false
```

## 10. 删除与回滚

删除策略：

- PostgreSQL 事务内先写 `access_blocks(reason=deleted)`。
- `documents.lifecycle_status = deleted`，`chunks.status = deleted`。
- `chunk_index_refs.visibility_state = blocked`。
- 尽力同步更新 Qdrant 和关键词索引 payload：`is_deleted=true`、`visibility_state=deleted`。
- 查询过滤条件始终包含 `is_deleted = false` 和 `visibility_state = active`，候选进入上下文前还必须检查 access block。
- 异步物理删除 Qdrant points 和关键词索引。
- 物理删除失败只影响存储成本，必须告警并重试，不能影响查询阻断。

回滚策略：

- 保留上一个可用 `index_version`。
- embedding 模型或维度变化时不能简单回滚，必须检查索引兼容性。
- Qdrant collection 可使用版本化 collection 或 payload version。

## 11. 权限索引刷新

权限变更后：

```text
1. Permission Service 更新 resource_policy
2. 创建 permission_index_refresh 任务
3. 对 strict mode 或高密级资源先写 access block 或 block 旧 chunk_index_refs
4. Indexing Service 生成新权限 payload 和 permission_snapshot
5. 更新 Qdrant payload
6. 更新关键词索引权限字段
7. 更新 chunks.permission_snapshot_id 和 chunk_index_refs.indexed_permission_version
8. 清理 access block 或解除 block
9. 写审计和指标
```

要求：

- strict mode 下权限 payload 刷新未完成前，旧权限用户不能通过旧索引或缓存访问。
- 查询发现 `indexed_permission_version < current_permission_version` 时必须拒绝候选或回源确认。
- 权限刷新失败必须保留阻断状态并告警。

## 12. 错误码

| 错误码 | 场景 |
| --- | --- |
| `INDEX_VECTOR_UNAVAILABLE` | 向量库不可用 |
| `INDEX_KEYWORD_UNAVAILABLE` | 关键词索引不可用 |
| `INDEX_DIMENSION_MISMATCH` | 向量维度不匹配 |
| `INDEX_PERMISSION_PAYLOAD_INVALID` | 权限字段缺失或非法 |
| `INDEX_PUBLISH_FAILED` | 索引发布失败 |
| `INDEX_ROLLBACK_FAILED` | 索引回滚失败 |
| `INDEX_VERSION_NOT_ACTIVE` | 查询命中非 active 索引 |
| `INDEX_PERMISSION_VERSION_STALE` | 索引权限版本落后 |
| `INDEX_VISIBILITY_BLOCKED` | 文档删除或权限收紧导致索引阻断 |

## 13. 测试点

- 向量索引和关键词索引都有同样权限字段。
- strict permission mode 下缺权限字段不能发布。
- 删除文档后查询不能命中。
- draft/ready 索引不能被查询命中。
- 权限索引刷新失败会告警。
- strict mode 下权限刷新失败时旧索引保持阻断。
- embedding 维度变化阻止错误发布。

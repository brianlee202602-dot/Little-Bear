# Auth Service 设计实现文档

## 1. 模块目标

Auth Service 负责企业内部最小生产阶段的本地账号认证，并为后续接入 LDAP、OIDC、SAML、企业微信、飞书等身份源预留扩展点。

## 2. 模块职责

- 本地账号登录。
- 密码强度校验和密码哈希。
- JWT 或服务端 session 签发与校验。
- 当前用户上下文加载。
- 管理员创建、禁用、重置账号。
- 服务间调用鉴权。
- 登录审计和账号操作审计。

## 3. 非职责

- 不计算资源访问权限，资源权限由 Permission Service 负责。
- 不管理业务配置。
- 不直接读取组织权限索引。
- 不决定用户能检索哪些 chunk。

## 4. 建议包结构

```text
app/modules/auth/
  service.py
  password_service.py
  session_service.py
  token_service.py
  user_repository.py
  policies.py
  schemas.py
  errors.py
  events.py
```

## 5. 核心表

`users`：

| 字段 | 说明 |
| --- | --- |
| id | 用户 ID |
| enterprise_id | 企业 ID |
| username | 登录账号 |
| password_hash | 密码哈希 |
| name | 姓名 |
| status | active、disabled、left |
| clearance_level | 用户密级 |
| attributes | 地区、岗位、职级等扩展属性 |
| created_at / updated_at | 时间戳 |

`sessions`，如果使用服务端 session：

| 字段 | 说明 |
| --- | --- |
| id | session ID |
| user_id | 用户 ID |
| status | active、revoked、expired |
| expires_at | 过期时间 |
| created_ip | 登录 IP |
| user_agent | User-Agent 摘要 |

`external_identities`，后续扩展：

| 字段 | 说明 |
| --- | --- |
| provider | ldap、oidc、saml、wecom、feishu |
| external_subject | 外部身份唯一标识 |
| user_id | 本地用户 ID |

## 6. 登录流程

```text
1. 接收 username/password
2. 校验系统 initialized
3. 读取 active auth config
4. 查询用户
5. 校验用户状态
6. 校验密码 hash
7. 检查登录失败次数和锁定状态
8. 签发 JWT 或创建 session
9. 写登录审计事件
10. 返回 access token
```

## 7. 密码策略

配置项：

- `auth.password_min_length`
- `auth.password_require_uppercase`
- `auth.password_require_lowercase`
- `auth.password_require_digit`
- `auth.password_require_symbol`
- `auth.login_failure_limit`
- `auth.lock_minutes`
- `auth.session_ttl_minutes`

实现要求：

- 密码使用 Argon2id 或 bcrypt。
- 密码明文只存在于请求生命周期内。
- 重置密码必须强制下次登录修改，或按企业策略立即生效。
- 登录失败次数写入 Redis 或数据库，必须有过期时间。

## 8. Token / Session

JWT claims 建议：

```json
{
  "sub": "user_123",
  "enterprise_id": "ent_001",
  "session_id": "sess_001",
  "auth_type": "local",
  "iat": 1777468800,
  "exp": 1777497600
}
```

JWT 中不放完整权限上下文。权限上下文由 Permission Service 按用户、组织、角色和权限版本实时构建或从缓存读取。

## 9. 管理 API

```http
POST /internal/v1/auth/login
POST /internal/v1/auth/logout
GET  /internal/v1/auth/me

POST /internal/v1/admin/users
GET  /internal/v1/admin/users
GET  /internal/v1/admin/users/{user_id}
PATCH /internal/v1/admin/users/{user_id}
POST /internal/v1/admin/users/{user_id}:disable
POST /internal/v1/admin/users/{user_id}:reset-password
```

管理员 API 必须先由 Auth Service 鉴别身份，再由 Permission Service 校验管理员权限。

## 10. 服务间认证

内部服务调用建议使用：

- mTLS，后续生产增强。
- 内部 JWT，最小阶段可用。
- API Key，仅限内部系统调用。

服务 token 不映射为普通用户，必须带明确 `service_name`、`scopes`、`expires_at`。

## 11. 错误码

| 错误码 | 场景 |
| --- | --- |
| `AUTH_INVALID_CREDENTIALS` | 用户名或密码错误 |
| `AUTH_USER_DISABLED` | 用户禁用 |
| `AUTH_SESSION_EXPIRED` | session 过期 |
| `AUTH_TOKEN_INVALID` | token 无效 |
| `AUTH_PASSWORD_WEAK` | 密码强度不足 |
| `AUTH_ACCOUNT_LOCKED` | 失败过多被锁定 |
| `AUTH_MFA_REQUIRED` | 后续扩展 MFA |

## 12. 审计事件

- `auth.login.succeeded`
- `auth.login.failed`
- `auth.logout`
- `auth.password.changed`
- `auth.password.reset`
- `auth.user.created`
- `auth.user.disabled`
- `auth.user.role.changed`

审计日志记录账号、操作者、来源 IP、结果和原因，不记录密码。

## 13. 测试点

- 正确账号密码登录成功。
- 错误密码增加失败次数。
- 达到失败限制后锁定。
- disabled 用户不能登录。
- JWT 过期不能访问业务 API。
- 管理员创建账号必须触发审计。
- 普通用户不能访问管理员账号 API。

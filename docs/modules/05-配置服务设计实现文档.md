# Config Service 设计实现文档

## 1. 模块目标

Config Service 负责初始化后的业务运行配置管理，包括配置读取、草稿、校验、发布、回滚、审批、热更新和审计。业务模块不得直接从环境变量读取运行配置。

## 2. 模块职责

- 管理 `active_config`。
- 管理配置草稿。
- 校验配置 schema、范围和依赖。
- 发布配置版本。
- 回滚配置版本。
- 高风险配置审批。
- 配置热更新通知。
- 提供按 scope 的有效配置解析。

## 3. 非职责

- 不管理 API 数据库连接启动配置。
- 不保存临时 `system_token`。
- 不直接执行查询或导入。
- 不替代 Secret Provider。

## 4. 建议包结构

```text
app/modules/config/

  schema_registry.py
  validator.py
  publisher.py
  resolver.py

```

## 5. 配置分层

| 层级 | 来源 | 说明 |
| --- | --- | --- |
| L0 | 启动配置 | 仅数据库连接配置 |
| L1 | active config | Redis、Secret Provider、MinIO、Qdrant、模型 provider 和普通运行配置 |
| L2 | active config + Secret Provider | 密钥和连接凭证 |
| L3 | active config + 审批 | 高风险策略配置 |
| L4 | 代码默认值 | 安全默认值和 schema 默认值 |

业务配置解析：

```text
knowledge_base 配置 > department 配置 > global 配置 > 代码默认值
```

配置项要求：

- 初始化 `active_config v1` 必须包含所有 `初始化必填=是` 的 global 配置。支持作用域覆盖的配置可以在初始化后再创建 department、knowledge_base、model_pool 或 connector 级配置。
- Secret value 不进入 `system_configs.value_json`，只保存 Secret 引用。
- 关键依赖地址、模型路由、权限、安全、缓存、限流和审计策略不得从环境变量读取。
- 代码默认值只允许作为 schema 默认值和安全兜底，不得替代初始化清单中的关键依赖配置。

## 8. 配置发布流程

```text
1. 管理员修改配置，保存为 draft
2. Config Service 按 schema 校验
3. 执行依赖连通性测试
4. 生成配置 diff 和风险等级
5. 高风险配置进入审批
6. 审批通过后发布 active version
7. 如果 Redis 已由当前 active config 初始化，则写 Redis 配置版本和变更通知
8. 各模块拉取新配置并热更新；Redis 不可用时退化为数据库轮询或进程内版本检查
9. 写审计日志
```

## 9. 高风险配置

必须审批：

- 部门权限策略。
- 企业公开默认策略或企业公开审批策略。
- Prompt 正文。
- Prompt 模板版本或引用策略。
- 模型路由到外部供应商。
- 是否记录查询原文。
- 是否记录完整 prompt。
- 数据保留周期缩短。
- embedding 模型或向量维度。
- 最终答案缓存开关或跨用户共享策略。
- 中文分词器、企业词库或同义词规则的大规模变更。

## 10. 运行时读取

推荐接口：

```python
class ConfigService:
    def get_active_version() -> int:
        ...

    def get_effective_config(
        key: str,
        scope_type: str = "global",
        scope_id: str | None = None
    ) -> ConfigValue:
        ...

    def get_config_bundle(scope: ConfigScope) -> ConfigBundle:
        ...
```

配置缓存：

- API 进程内存缓存 active config。
- Redis 在 ServiceBootstrap 成功后保存当前版本和变更通知。
- 收到配置变更后增量刷新或整体刷新。
- 配置读取失败时使用本地最近一次 active config 只适用于已成功启动后的运行期；冷启动时数据库不可用必须失败。

## 12. 错误码

| 错误码 | 场景 |
| --- | --- |
| `CONFIG_NOT_INITIALIZED` | 未初始化不能读取业务配置 |
| `CONFIG_ACTIVE_MISSING` | initialized=true 但 active config 缺失 |
| `CONFIG_SCHEMA_INVALID` | schema 校验失败 |
| `CONFIG_DEPENDENCY_FAILED` | 依赖连通性测试失败 |
| `CONFIG_APPROVAL_REQUIRED` | 高风险配置需要审批 |
| `CONFIG_VERSION_CONFLICT` | 配置版本冲突 |
| `CONFIG_ROLLBACK_UNSAFE` | 回滚不兼容 |

## 14. 测试点

- 业务模块不能绕过 Config Service。
- 配置发布会生成版本。
- 高风险配置未审批不能发布。
- 配置回滚做兼容性检查。
- active config 不存在时拒绝服务。
- Secret value 不会进入 API 响应和普通日志。
- 高风险缓存策略和 Prompt 记录策略未经审批不能发布。

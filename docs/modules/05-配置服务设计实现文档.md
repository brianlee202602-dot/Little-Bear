# Config Service 设计实现文档

## 1. 模块目标

Config Service 负责初始化后的业务运行配置管理，包括配置读取、草稿、校验、发布、回滚、审批、热更新和审计。业务模块不得直接从环境变量读取运行配置。

## 2. 模块职责

- 管理 `active_config`。
- 管理配置草稿。
- 校验配置 schema、范围和依赖。
- 发布配置版本。
- 回滚配置版本。
- 高风险配置审批。
- 配置热更新通知。
- 提供按 scope 的有效配置解析。

## 3. 非职责

- 不管理 API 启动环境变量。
- 不保存临时 `system_token`。
- 不直接执行查询或导入。
- 不替代 Secret Provider。

## 5. 配置分层

| 层级 | 来源 | 说明 |
| --- | --- | --- |
| L0 | 环境变量 | API 启动配置 |
| L1 | Secret Provider | 密钥和连接凭证 |
| L2 | active config | 普通运行配置 |
| L3 | active config + 审批 | 高风险策略配置 |
| L4 | 代码默认值 | 安全默认值和 schema 默认值 |

业务配置解析：

```text
knowledge_base 配置 > department 配置 > global 配置 > 代码默认值
```

## 6. 核心表

`system_configs`：

| 字段 | 说明 |
| --- | --- |
| id | 配置记录 ID |
| key | 配置 key |
| value_json | 配置值 |
| schema_json | 配置 schema |
| scope_type | global、department、knowledge_base、model_pool、connector |
| scope_id | 作用域 ID |
| status | draft、active、archived |
| version | 配置版本 |

`config_versions`：

| 字段 | 说明 |
| --- | --- |
| version | 版本号 |
| status | draft、pending_approval、active、rolled_back |
| diff_json | 配置 diff |
| risk_level | low、medium、high |
| created_by | 创建人 |
| approved_by | 审批人 |
| published_at | 发布时间 |

`config_change_events`：

| 字段 | 说明 |
| --- | --- |
| id | 事件 ID |
| config_version | 配置版本 |
| event_type | published、rolled_back |
| consumed_by | 消费服务 |
| consumed_at | 消费时间 |

## 8. 配置发布流程

```text
1. 管理员修改配置，保存为 draft
2. Config Service 按 schema 校验
3. 执行依赖连通性测试
4. 生成配置 diff 和风险等级
5. 高风险配置进入审批
6. 审批通过后发布 active version
7. 写 Redis 配置版本和变更通知
8. 各模块拉取新配置并热更新
9. 写审计日志
```

## 9. 高风险配置

必须审批：

- 部门权限策略。
- 高密级知识库可见范围。
- Prompt 正文。
- Prompt 模板版本或引用策略。
- 模型路由到外部供应商。
- 是否记录查询原文。
- 是否记录完整 prompt。
- 数据保留周期缩短。
- embedding 模型或向量维度。
- 最终答案缓存开关、跨用户共享策略或高密级缓存策略。
- 中文分词器、企业词库或同义词规则的大规模变更。

## 10. 运行时读取

推荐接口：

```python
class ConfigService:
    def get_active_version() -> int:
        ...

    def get_effective_config(
        key: str,
        scope_type: str = "global",
        scope_id: str | None = None
    ) -> ConfigValue:
        ...

    def get_config_bundle(scope: ConfigScope) -> ConfigBundle:
        ...
```

配置缓存：

- API 进程内存缓存 active config。
- Redis 保存当前版本和变更通知。
- 收到配置变更后增量刷新或整体刷新。
- 配置读取失败时使用本地最近一次 active config。

## 11. API

```http
GET  /internal/v1/admin/configs
GET  /internal/v1/admin/configs/{key}
PUT  /internal/v1/admin/configs/{key}
POST /internal/v1/admin/configs:validate
POST /internal/v1/admin/configs:publish
POST /internal/v1/admin/configs:rollback
GET  /internal/v1/admin/configs/versions
GET  /internal/v1/admin/configs/diff?from=12&to=13
```

写接口只允许系统管理员或安全管理员访问。

## 12. 错误码

| 错误码 | 场景 |
| --- | --- |
| `CONFIG_NOT_INITIALIZED` | 未初始化不能读取业务配置 |
| `CONFIG_ACTIVE_MISSING` | initialized=true 但 active config 缺失 |
| `CONFIG_SCHEMA_INVALID` | schema 校验失败 |
| `CONFIG_DEPENDENCY_FAILED` | 依赖连通性测试失败 |
| `CONFIG_APPROVAL_REQUIRED` | 高风险配置需要审批 |
| `CONFIG_VERSION_CONFLICT` | 配置版本冲突 |
| `CONFIG_ROLLBACK_UNSAFE` | 回滚不兼容 |

## 13. 审计事件

- `config.draft.saved`
- `config.validate.failed`
- `config.approval.requested`
- `config.approval.approved`
- `config.published`
- `config.rollback`
- `config.hot_reload.failed`

## 14. 测试点

- 业务模块不能绕过 Config Service。
- 配置发布会生成版本。
- 高风险配置未审批不能发布。
- 配置回滚做兼容性检查。
- active config 不存在时拒绝服务。
- Secret value 不会进入 API 响应和普通日志。
- 高风险缓存策略和 Prompt 记录策略未经审批不能发布。

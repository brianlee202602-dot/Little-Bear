# Import Service 与 Workers 设计实现文档

## 1. 模块目标

Import Service 负责文档导入入口和任务创建，Import Workers 负责异步执行解析、清洗、切块、向量化、索引写入和发布，保证大批量导入不阻塞线上查询。

## 2. 模块职责

Import Service：

- 校验导入权限。
- 校验文件大小、类型、幂等键。
- 创建 `import_jobs`、`documents`。
- 原始文件写入对象存储。
- 返回 `job_id`。
- 提供任务状态查询。

Import Workers：

- 领取导入任务。
- 执行阶段状态机。
- 解析、清洗、切块。
- 调用 embedding。
- 写向量索引和关键词索引。
- 校验索引可用性。
- 发布索引版本。
- 失败重试和补偿清理。

## 3. 非职责

- 不直接决定用户权限，导入权限由 Permission Service 校验。
- 不直接调用 Qdrant、MinIO、模型 SDK，必须通过 ports/adapters。
- 不在 HTTP 请求内执行长耗时解析和 embedding。

## 5. 核心表

`import_jobs`：

| 字段 | 说明 |
| --- | --- |
| id | 任务 ID |
| enterprise_id | 企业 ID |
| kb_id | 知识库 ID |
| created_by | 创建人 |
| job_type | upload、api_batch、connector_sync |
| status | queued、running、retrying、partial_success、success、failed、cancelled |
| stage | validate、parse、clean、chunk、embed、index、publish |
| total_items | 总数 |
| succeeded_items | 成功数 |
| failed_items | 失败数 |
| error_summary | 错误摘要 |
| idempotency_key | 幂等键 |
| priority | 优先级 |
| locked_by | Worker ID |
| locked_until | 锁过期时间 |
| retry_count | 重试次数 |
| next_retry_at | 下次重试时间 |

`documents`、`document_versions`、`chunks`、`index_versions` 和 `chunk_index_refs` 由核心数据模型统一维护，字段契约详见 `14-核心数据模型设计实现文档.md`。

## 6. 导入 API

```http
POST /internal/v1/knowledge-bases/{kb_id}/documents:upload
POST /internal/v1/knowledge-bases/{kb_id}/documents:import
GET  /internal/v1/import-jobs/{job_id}
POST /internal/v1/import-jobs/{job_id}:cancel
POST /internal/v1/import-jobs/{job_id}:retry
```

HTTP 导入响应：

```json
{
  "job_id": "job_001",
  "status": "queued",
  "document_ids": ["doc_001"]
}
```

## 7. 总流程

```text
1. API Gateway 做认证、限流、请求大小检查
2. Permission Service 校验 document:import 和目标知识库写权限
3. Import Service 校验文件类型、大小、幂等键
4. 写 import_job 和 document
5. 原始文件写入 ObjectStorage
6. Worker 领取任务
7. validate
8. parse
9. clean
10. chunk
11. embed
12. index_vector(draft)
13. index_keyword(draft)
14. validate_index，确认 draft 索引对查询不可见
15. publish，激活 active index version
16. 写审计和指标
```

## 8. Worker 领取任务

基于 PostgreSQL 任务表：

```text
1. 查询 queued/retrying 且 next_retry_at <= now 的任务
2. 使用 SKIP LOCKED 或乐观锁领取
3. 设置 locked_by、locked_until、status=running
4. 执行当前 stage
5. 成功后推进 stage
6. 失败可重试则 retry_count + 1，status=retrying，设置 next_retry_at
7. 不可重试则 failed
```

Worker 必须定期续约 `locked_until`。如果 Worker 崩溃，锁过期后其他 Worker 可接管。

状态推进约束：

- `queued -> running -> success` 表示首次执行成功。
- `running -> retrying -> running` 表示可重试失败后的再次领取。
- `running -> failed` 只用于不可重试错误或超过最大重试次数。
- `cancelled` 只能由 API 或管理员动作触发，Worker 领取和阶段推进前必须检查取消状态。
- 发布阶段成功前，`document.index_status` 不能变为 `indexed`，`chunk_index_refs.visibility_state` 必须保持 `draft`。

## 9. 解析阶段

统一解析输出：

```json
{
  "doc_id": "doc_001",
  "title": "销售合同审批制度",
  "sections": [
    {
      "heading": "审批材料",
      "level": 2,
      "text": "销售合同审批需提交...",
      "page_start": 3,
      "page_end": 4,
      "tables": [],
      "images": [],
      "source_offsets": {"start": 1200, "end": 1980}
    }
  ],
  "metadata": {
    "author": "alice",
    "department": "dept_sales"
  }
}
```

解析要求：

- 保留页码、标题层级、表格、图片说明。
- 扫描件走 OCR，并记录 OCR 置信度。
- 代码、配置、公式、合同条款不能被普通清洗破坏。
- 失败记录解析器版本和错误码。

## 10. 清洗与切块

清洗原则：提高检索质量，不改写业务事实。

切块默认策略：

- 按标题、段落、语义边界切分。
- chunk 大小 300 到 800 tokens。
- overlap 50 到 150 tokens。
- 保留 heading_path、页码、文档版本、来源 URI。
- FAQ 问答对、合同条款、表格尽量不拆散。

## 11. Embedding 阶段

要求：

- 使用 Model Gateway。
- document embedding 使用离线 batch 池。
- 向量 ID 确定性生成：`chunk_id + embedding_model + doc_version`。
- 记录 `embedding_model`、`model_version`、`dimension`。
- 模型失败可重试，超过阈值进入 failed 或 dead-letter。

## 12. 并发控制

建议值：

| 维度 | 建议 |
| --- | --- |
| 全局导入任务并发 | 3 到 5 |
| 单部门导入任务并发 | 1 到 2 |
| 单用户导入任务并发 | 1 |
| 单任务文件并发 | 2 到 4 |
| Embedding batch size | 16 到 64 |
| index batch size | 100 到 500 chunks |

并发策略由 active config 管理。

## 13. 错误码

| 错误码 | 场景 |
| --- | --- |
| `IMPORT_FILE_TOO_LARGE` | 文件过大 |
| `IMPORT_FILE_TYPE_UNSUPPORTED` | 文件类型不支持 |
| `IMPORT_IDEMPOTENCY_CONFLICT` | 幂等键冲突 |
| `IMPORT_PARSE_FAILED` | 文档解析失败 |
| `IMPORT_CLEAN_EMPTY` | 清洗结果为空 |
| `IMPORT_CHUNK_FAILED` | 切块失败 |
| `IMPORT_EMBED_FAILED` | 向量化失败 |
| `IMPORT_CANCELLED` | 用户取消 |

## 14. 审计事件

- `import.job.created`
- `import.job.cancelled`
- `import.stage.started`
- `import.stage.succeeded`
- `import.stage.failed`
- `import.job.completed`

## 15. 测试点

- HTTP 上传只创建任务，不执行长耗时导入。
- 同一幂等键重复提交返回同一任务。
- Worker 崩溃后任务可被其他 Worker 接管。
- 解析失败可定位文件和解析器版本。
- embedding 服务不可用不会影响查询链路。
- 导入失败不会发布未完成或未校验的索引。
- draft 索引写入后在发布前不会被查询命中。

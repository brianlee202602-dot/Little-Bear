# Audit 与 Observability 设计实现文档

## 1. 模块目标

Audit 与 Observability 负责让系统关键行为可追踪、可审计、可告警、可评测。企业内部 RAG 不仅要回答问题，还要能解释“谁在什么时候访问了什么资料、为什么能访问、系统是否降级、引用是否可靠”。

## 2. 模块职责

Audit：

- 查询审计。
- 导入审计。
- 权限变更审计。
- 配置发布审计。
- 初始化审计。
- 登录和管理员操作审计。
- 高风险模型调用审计。

Observability：

- 结构化日志。
- Metrics。
- Trace。
- 慢查询分析。
- 导入任务监控。
- 模型服务监控。
- 质量评测事件。

## 3. 非职责

- 不负责业务权限决策。
- 不保存文档全文作为普通日志。
- 不替代数据备份。

## 5. 结构化日志

所有服务输出 JSON 日志：

```json
{
  "timestamp": "2026-04-29T20:00:00+08:00",
  "level": "INFO",
  "service": "rag-api",
  "module": "query",
  "request_id": "req_001",
  "trace_id": "trace_001",
  "user_id": "user_123",
  "enterprise_id": "ent_001",
  "config_version": 12,
  "permission_version": 42,
  "index_version_hash": "idxh_001",
  "stage": "rerank",
  "latency_ms": 238,
  "status": "success",
  "degraded": false
}
```

禁止进入普通日志：

- 密码。
- token。
- secret value。
- 完整 prompt。
- 未脱敏个人信息。
- 大段文档原文。

Prompt 相关日志默认只记录模板 ID、模板版本、prompt hash、变量摘要和模型版本；完整 prompt 只能进入受控审计存储。

## 7. Metrics

查询指标：

- QPS。
- P50/P95/P99 延迟。
- 各阶段延迟。
- 权限过滤数量。
- 向量召回命中数。
- 关键词召回命中数。
- rerank 超时率。
- LLM 首 token 延迟。
- LLM 总延迟。
- 降级率。
- 无可访问资料比例。

导入指标：

- 导入任务数量。
- 各阶段成功率。
- 各阶段失败率。
- Worker 队列积压。
- 平均 chunk 数。
- Embedding tokens/s。
- 索引写入耗时。

模型指标：

- embedding latency。
- rerank latency。
- LLM first token latency。
- token usage。
- GPU pool overloaded 次数。
- 熔断次数。

## 8. 审计事件模型

`audit_logs`：

| 字段 | 说明 |
| --- | --- |
| id | 审计 ID |
| enterprise_id | 企业 ID |
| actor_type | user、service、system |
| actor_id | 操作者 ID |
| action | 操作 |
| resource_type | query、document、permission、config、setup |
| resource_id | 资源 ID |
| request_id | 请求 ID |
| trace_id | Trace ID |
| result | success、failed、denied |
| risk_level | low、medium、high |
| summary_json | 摘要 |
| created_at | 时间 |

## 9. 必须审计的操作

- 首次初始化 validate/initialize。
- system token 创建和轮换。
- 登录成功和失败。
- 用户创建、禁用、重置密码。
- 权限策略变更。
- 查询命中的文档和 chunk 摘要。
- 查询使用的 `permission_filter_hash`、`index_version_hash` 和 `cache_key_hash`。
- access block 命中、索引权限版本落后和缓存命中后的候选阻断。
- 导入任务创建、发布、失败。
- 配置发布、审批、回滚。
- 企业公开文档访问和权限变更。
- 模型路由到外部供应商。

## 10. 脱敏策略

Audit Redactor 负责：

- 密码字段删除。
- token 字段 hash 化。
- secret value 替换为 secret ref。
- 查询文本按配置存储 hash 或明文。
- 文档 snippet 限长。
- 个人敏感信息打标或脱敏。
- Prompt 默认只保留模板 ID、模板版本、prompt hash、变量摘要和模型版本。

## 11. 质量评测

离线评测集字段：

- query。
- 标准答案。
- 期望引用文档。
- 用户角色和权限上下文。
- 禁止命中文档。
- 业务标签。

核心指标：

- Recall@K。
- MRR。
- NDCG。
- Citation Accuracy。
- Answer Groundedness。
- Permission Violation Rate。
- Cache Permission Isolation Rate。
- Draft Index Exposure Count。
- P95 latency。

## 12. 告警建议

- `Permission Violation Rate > 0` 立即告警。
- active config 加载失败。
- initialized=true 但 active config 缺失。
- draft index 被查询命中。
- 查询缓存发生跨用户或跨权限过滤条件复用。
- 索引权限版本落后且未阻断。
- 导入失败率持续升高。
- Worker 队列积压超过阈值。
- Qdrant 或 PostgreSQL Full Text 不可用。
- LLM 超时率升高。
- 审计写入失败。

## 13. 错误码

| 错误码 | 场景 |
| --- | --- |
| `AUDIT_WRITE_FAILED` | 审计写入失败 |
| `OBS_TRACE_CONTEXT_MISSING` | Trace 上下文缺失 |
| `OBS_METRIC_WRITE_FAILED` | 指标写入失败 |
| `EVAL_DATASET_INVALID` | 评测集非法 |

## 14. 测试点

- 查询、导入、权限变更都有审计。
- 普通日志不包含 secret 和 token。
- 普通日志不包含完整 prompt、文档原文和未脱敏个人信息。
- request_id 贯穿所有阶段。
- 审计写入失败不能静默丢失。
- 高风险操作有 risk_level。
- 评测能检测权限越权。
- 评测能检测缓存串权和 draft 索引可见问题。

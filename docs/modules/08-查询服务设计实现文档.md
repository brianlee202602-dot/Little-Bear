# Query Service 设计实现文档

## 1. 模块目标

Query Service 负责查询 API 的入口编排，包括认证后的请求上下文、参数校验、权限上下文、限流、超时预算、流式响应、日志和错误处理。

## 2. 模块职责

- 提供普通查询和流式查询 API。
- 校验查询参数。
- 构建 request context。
- 调用 Permission Service 构建权限上下文。
- 获取 active config 和 active index versions。
- 调用 Retrieval/Context/Answer 编排。
- 对召回候选执行 metadata/access block/citation gate。
- 处理降级响应。
- 写查询日志、指标、审计。

## 3. 非职责

- 不直接访问 Qdrant、MinIO、模型 SDK。
- 不直接计算复杂权限策略。
- 不负责文档导入。

## 5. API

```http
POST /internal/v1/query
POST /internal/v1/query:stream
```

请求示例：

```json
{
  "kb_ids": ["kb_sales", "kb_policy"],
  "query": "销售合同审批需要哪些材料？",
  "mode": "answer",
  "filters": {
    "department_scope": "my_accessible",
    "updated_after": "2026-01-01",
    "source_type": ["policy", "faq"],
    "tags": ["合同"]
  },
  "top_k": 8,
  "stream": true,
  "include_sources": true
}
```

响应示例：

```json
{
  "request_id": "req_001",
  "answer": "销售合同审批通常需要合同正文、客户主体资料、报价单和审批单。",
  "citations": [

      "page": "3-4",
      "score": 0.92
    }
  ],
  "confidence": "medium",
  "degraded": false,

}
```

## 6. 查询入口流程

```text
1. Setup guard 检查 initialized
2. Auth Service 校验用户
3. Rate Limit 检查
4. Query Service 校验请求
5. Config Service 获取 active config
6. Indexing Service 或数据模型仓库获取 active index versions
7. Permission Service 构建权限上下文和过滤条件
8. RetrievalOrchestrator 执行检索编排
9. Candidate Gate 校验文档状态、access block、active index 和权限快照
10. Context Service 组装上下文
11. Answer Service 生成答案
12. Citation Verifier 校验引用
13. Safety Filter 后处理
14. 写查询日志、审计、指标
15. 返回响应
```

## 8. 超时预算

示例：

| 阶段 | 预算 |
| --- | --- |
| 权限上下文 | 50ms |
| rewrite | 300ms |
| embedding | 300ms |
| 多路召回 | 500ms |
| rerank | 800ms |
| context | 100ms |
| LLM 首 token | 3000ms |
| 总请求 | 8000ms |

超时后按降级策略继续或返回结构化错误。

## 9. 流式响应

SSE 事件建议：

```text
event: metadata
event: token
event: citation
event: done
event: error
```

要求：

- 流式响应开始前必须完成权限过滤。
- 流式响应开始前必须完成 active index、access block 和候选 citation gate。
- token 输出期间仍需支持取消请求。
- 最终 done 事件带引用和统计信息。
- 如果引用校验失败，不能只流出无引用答案。

严格引用模式下的流式策略：

- 默认策略是先生成到服务端缓冲区，完成引用校验后再开始输出 token。
- 如果业务必须边生成边输出，只能输出草稿事件 `event: draft_token`，最终 `done` 前必须完成引用校验；客户端不得把草稿当成合规最终答案。
- 对所有查询，禁止输出未经引用校验的最终 token。

## 11. 错误码

| 错误码 | 场景 |
| --- | --- |
| `QUERY_INVALID_REQUEST` | 参数非法 |
| `QUERY_TOO_LONG` | 查询文本过长 |
| `QUERY_RATE_LIMITED` | 限流 |
| `QUERY_TIMEOUT` | 总超时 |
| `QUERY_CANCELLED` | 用户取消 |
| `RAG_NO_ACCESSIBLE_CONTEXT` | 没有可访问资料 |
| `RAG_CITATION_VERIFY_FAILED` | 引用校验失败 |
| `RAG_CANDIDATE_BLOCKED` | 候选被删除、阻断或索引版本不可见 |

## 12. 测试点

- 未初始化时返回 `SETUP_REQUIRED`。
- 普通用户不能查询无权限知识库。
- 流式输出也必须带引用。
- 流式响应不会输出未经引用校验的最终答案。
- draft 索引和已删除文档不能进入上下文。
- 查询缓存命中后仍会执行 access block 和引用有效性 gate。
- embedding 失败时降级关键词检索。
- LLM 不可用时返回检索结果和引用。
- 每次查询都记录 `config_version` 和 `permission_version`。
